C-2-2. **`select_pln_mixture_local_nb.py` (637 lines)** — pure-Python staged selector. Module docstring (L4-12): *"Architecture (non-alias, staged): Stage A: build multiple local prior experts from neighbor bins. Stage B: pool normalized local experts via evidence add (hplus semantics). Stage C: tensor-compose pooled prior with likelihood-only NB evidence. The ranking signal is posterior strength: strength = post_pos / (post_pos + post_neg). This script is intentionally pure-Python for speed and auditability."*

C-2-3. **Input format (verified L70-115):** Per-problem TPTP `.p` file plus a `.vec` file of sparse feature lines (`idx:val ...`); `read_problem_formulas` parses formulas with roles (`axiom|hypothesis|definition` vs `conjecture|negated_conjecture`); `parse_sparse_line` returns a set of feature indices. Dataset path is hardcoded to `/home/zar/claude/atps/datasets/extended_mptp5k` (Linux/zar's machine; will not run as-is on this Windows checkout — see §10).

C-2-4. **Scoring algorithm (verified L209-281, L306-371, L374-481):** (a) `find_neighbors` does IDF-weighted feature overlap to find top-K training neighbors; (b) `local_prior_by_mixture_bins` partitions neighbors into bins `[4,16,64,256]` (cumulative cutoffs), and for each bin computes `pos_mass[ax] += sim` if ax used by neighbor and similarly weighted-negative `neg_mass = (total_mass - pos_raw) * neg_weight` (default `neg_weight=0.08`) — i.e., weak-negative kNN evidence. Each bin is normalized to a target total by `normalize_to_total` (preserves strength). Bins are pooled via direct addition: `p_sum += np; n_sum += nn` over bins (this IS evidence-hplus per L70-79). (c) `add_global_prior` optionally adds a global tfreq-derived prior expert. (d) `likelihood_evidence_by_nb_scores` invokes `mash_nb_score` with `sigma1=0.0` (prior term removed) per ax, IQR-scales raw NB log-scores, converts to odds via `s = exp(scaled)/(exp(scaled)+1)`, normalizes to fixed `t_lik`. (e) Tensor composition L453-456: `post_pos = ppos * lpos; post_neg = pneg * lneg`. (f) Strength projection L82-86: `pos / (pos + neg)`. Final rank optionally adds `support_tiebreak * support` where `support = (ppos+pneg)/((ppos+pneg) + support_kappa)`.

The architectural note at L13 ("intentionally pure-Python for speed and auditability") matches the headline paper L670 ("Pure-Python selector: same evidence algebra, staged explicitly for auditability/performance"). The Lean theorems are paper-side correspondences; no runtime check at execution time.

C-3-3. **HEADLINE THEOREM 1 — `toStrength_nbEvidence_eq_nbPosterior` (verified VERBATIM at `Mettapedia/Logic/PLNBayesNetInference.lean` L290-298):**
```
theorem toStrength_nbEvidence_eq_nbPosterior
    [Fintype ι]
    (prior : BinaryEvidence) (likelihood : ι → BinaryEvidence) :
    BinaryEvidence.toStrength (nbEvidence prior likelihood) =
      nbPosterior prior.pos prior.neg
        (fun i => (likelihood i).pos) (fun i => (likelihood i).neg) := by
  classical
  unfold BinaryEvidence.toStrength nbPosterior
  simp [BinaryEvidence.total, nbEvidence_pos, nbEvidence_neg]
```
Statement: *the PLN strength view of NB evidence equals the standard NB posterior.* No `sorry`; full proof. There is also a non-breaking alias `PLN_tensorStrength_eq_nbPosterior` (L322-328) defined as `exact toStrength_nbEvidence_eq_nbPosterior`. The math sketch (L275-289) makes the claim explicit: with prior=(p+,p−) and likelihood_i=(li+,li−), `nbEvidence prior likelihood = (p+ * Π_i li+, p− * Π_i li−)` and PLN strength = `(p+ Π li+) / ((p+ Π li+) + (p− Π li−))` = `nbPosterior`.

C-3-4. **HEADLINE THEOREM 2 — `PLN_hplusPos_eq_knnRelevance` (verified VERBATIM at `Mettapedia/Logic/PremiseSelectionKNN_PLNBridge.lean` L125-130):**
```
theorem PLN_hplusPos_eq_knnRelevance {Fact : Type*} [DecidableEq Fact]
    (goal : Fact) (N : Finset Fact) (near : Fact -> Fact -> ℝ≥0∞)
    (deps : DepSet Fact) (tau2 : ℝ≥0∞) (phi : Fact) :
    (plnKnnEvidence goal N near deps tau2 phi).pos =
      knnRelevanceENN goal N near deps tau2 phi := by
  exact plnKnnEvidence_pos_eq_knnRelevanceENN goal N near deps tau2 phi
```
Statement: *the positive-evidence channel of the PLN k-NN evidence aggregator equals the standard MaSh-style k-NN relevance score.* L132-139 also proves `plnKnnEvidence_neg_eq_zero` (the negative channel of pure positive transfer is zero by `posEvidence(w) = (w, 0)`).
