`pln-experimental2` is **Nil Geisweiller's personal research playground** (471 of ~520 commits, ~95% of repo). Primary deliverables are the **`docs/nuPLN/nuPLN.tex` manuscript** (nuPLN = "new universal PLN"), supporting use-case docs (surprisingness, next-token-prediction), Maxima symbolic derivations (conjunction-introduction), and a narrow `experimental/early-pln/` 4-encoding experiment (dependent-types / entail / equal / match) covering only 3 rules.

**One new rule worth adding to wiki coverage**: **ImplicationDirectIntroduction (IDI)** — base + recursive Bayesian forms (`inc-pos-count` / `inc-neg-count`) with sorted evidence IDs for double-counting prevention. Not present in Sources 2-3.

## 2. Unanimous cross-model findings

### 2.1 [X3] nuPLN is a real successor-theory attempt with a Solomonoff-style grounding

All three models confirm: `docs/nuPLN/nuPLN.tex` introduces a **global universal distribution ν** to ground induction, mirroring Solomonoff induction. Motivation: fix the 2008 book's acknowledged inability to globally rescue induction via any single universe-size choice (Source 1 §2.8). **ν is uncomputable in the strict Solomonoff sense**; nuPLN treats it as a theoretical anchor rather than a computable primitive.

**[X1 Claude] Methodological pivot within nuPLN**: nuPLN **dissolves the intensional/extensional split** rather than implementing conversion rules between them. It keeps only the extensional side via an indicator-function / satisfying-set isomorphism (Claude: L254-266 verbatim — extensional = induction-approach to inheritance, intensional = abduction-approach to the same inheritance). If accepted, this would obsolete an entire book-era machinery. Worth expert review.

**[X3] Derivations are incomplete**: §4.1 (Predicate Direct Introduction) and §4.2 (Implication Direct Introduction) both end with bare `NEXT` placeholders. 6 un-resolved `\cite{TODO}`. One empty section, one empty subsection. **Readers should treat nuPLN as proposed-theory, not complete-theory.**

### 2.2 [X3 CORRECTED] Bl / Pr are NOT the missing TV hierarchy

Gemini's original preview hypothesis (before extraction) was that Bl and Pr might be Indefinite/Distributional TVs. **All three extractions agree: they are NOT.**

- `Bl` wraps `Bool` with count = 1e9 (fixed certainty).
- `Pr` wraps `Number` with count = 1e9 (fixed certainty).
- Claude verified: **no rule formula consumes Bl or Pr** — only STV is destructured by Deduction / ModusPonens / IDI.
- They are degenerate high-confidence wrappers, not evidence-accumulation types.

The book's IndefiniteTV (`<[L,U], b, k>`) and DistributionalTV (StepFunction, Polynomial) remain **unimplemented** in this repo, same as in Sources 2-3.

### 2.3 [X3] Confidence-pivot status: PREDATES pln0.9

All three models confirm `pln-experimental2` uses `min`-based / count-based confidence formulas, matching PLN3 and the pre-pivot state. **The pln0.9 multiplicative-confidence pivot is NOT present here.**

### 2.4 [X3] Rule-family recovery: effectively zero

Only 3 rules implemented across all four `experimental/early-pln/{dependent-types, entail, equal, match}/` encodings:
- Deduction
- ModusPonens
- ImplicationDirectIntroduction (**NEW — not in Sources 2-3**)

**All confirmed absent**: quantifiers (ForAll/ThereExists), temporal/causal (PredictiveImplication/SeqAND family), intensional↔extensional conversion rules, distributional / indefinite TV machinery, concept-geometry deduction, multideduction, BRGI, OBI, attention/importance values, FactorGraph, MORK, Context as first-class, possible-worlds variable instantiation.

### 2.5 [X3] ImplicationDirectIntroduction (IDI) — the one new rule

Distinct from Sources 2-3. Gemini's concise framing (verified by Claude's detail):
- **IDIBase**: base case for empty evidence set.
- **IDIRecursive**: accumulates evidence via sorted-ID check to prevent double-counting when composing.
- Helper functions: `inc-pos-count`, `inc-neg-count` — incremental Bayesian updates.

Interpretation: IDI is the rule that would generate an Implication link's truth-value from direct observation of the implication holding / failing, rather than from deduction of component terms. Candidate to propose for the wiki's `Hyperon AI Algorithms+PLN+PLN Full` as a "forthcoming rule family" once Geisweiller completes the nuPLN derivations.

### 2.6 [X3] Maxima `.mac` is exploratory, not yet ported

`experimental/truth-value-formula/conjunction-introduction/conjunction-xp.mac` (387 lines) derives `Pr(pAB = x)` as a Stirling-approximation of the binomial-combination probability for conjunction-introduction. **Key theoretical result (Claude verified verbatim)**: under independence assumption, the distribution converges to a **Dirac delta at pA·pB as n → ∞**.

Also studies A→B-dependency and A↔B-dependency variants. Conclusion in the file: *"the structure of a composite predicate, by itself, provides some forms of evidence."*

**Status**: exploratory, **not ported into any rule formula in the MeTTa tree**. The repo's code and the Maxima work are parallel research tracks, not integrated.

### 2.7 [X3] Repo activity status

Active research phase ran 2025-06 through 2025-12 (nuPLN manuscript, surprisingness doc, next-token-prediction use case, conjunction-introduction). **No code/theory changes in the last 3 months** — only "More brainstorming notes" + `Misc`. Repo is in an **introspective / reflective phase**, not mid-implementation-sprint.
