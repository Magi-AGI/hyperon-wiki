## 2. `docs/nuPLN/` - theoretical manuscript

C-2-1: `nuPLN.tex` explicitly proposes a Solomonoff-style reformulation of PLN. The introduction says the goal is to approach an unknown first-order distribution `mu` via a known second-order universal distribution `nu`, and because this "departs somewhat from the definition of PLN in the PLN book" the author gives it a new name, `nuPLN`. [docs/nuPLN/nuPLN.tex:L55-L79]

C-2-2: The manuscript's core semantic move is to make a global probability space unavoidable. In the Definitions section, the author says he "cannot conceive a complete definition of PLN that does not rely on such global probability distribution", then defines `(Omega, Field, nu)` where each world contains a probabilistic predicate/program plus its full mapping over the domain. [docs/nuPLN/nuPLN.tex:L81-L101] [docs/nuPLN/nuPLN.tex:L102-L170]

C-2-3: The global-distribution proposal is stronger than a notational cleanup. `mu` is typed as `Domain -> Omega -> Bool` (or curried multi-argument variants), and the manuscript treats reasoning about unknown worlds/generators as fundamental rather than optional. That is the repo's clearest attempt to answer the classic PLN problem of induction without a single fixed global event space. [docs/nuPLN/nuPLN.tex:L130-L167] [docs/nuPLN/nuPLN.tex:L222-L228]

C-2-4: The manuscript does not implement the original intensional/extensional split; it argues that the split should disappear conceptually. In `Concepts vs Predicates`, Geisweiller says extensional inheritance is induction-oriented, intensional inheritance is abduction-oriented, and either side can define the other, so he chooses the extensional side and then recovers the predicate side through indicator-function / satisfying-set isomorphism. [docs/nuPLN/nuPLN.tex:L241-L266] [docs/nuPLN/nuPLN.tex:L295-L315]

C-2-5: The manuscript is still incomplete at source level. It contains unresolved `\cite{TODO}` placeholders, unresolved `\ref{TODO}` placeholders, and bare `NEXT` markers in both the Bernoulli-posterior subsection and the Implication Direct Introduction subsection. Two subsection headers, `Truth Value as Posterior Distribution` and `Statement versus Judgement`, are effectively empty. [docs/nuPLN/nuPLN.tex:L56-L89] [docs/nuPLN/nuPLN.tex:L213-L213] [docs/nuPLN/nuPLN.tex:L256-L291] [docs/nuPLN/nuPLN.tex:L336-L339] [docs/nuPLN/nuPLN.tex:L384-L411]

C-2-6: The Implication Direct Introduction derivation is specifically unfinished. The text says the rule is not explicit in the 2008 book but can be derived from iterative induction and revision, and it sets up a Beta/Bernoulli posterior story for implication truth values; then the actual derivation stops at `NEXT`. [docs/nuPLN/nuPLN.tex:L392-L411]

C-2-7: The practical consequence is that `nuPLN.tex` is a successor-theory draft, not a completed theoretical replacement for the 2008 book. Its aspiration is clear and important, but the repo does not contain finished proofs or a completed semantics document that could be cited as settled doctrine without expert review. [docs/nuPLN/nuPLN.tex:L51-L52] [docs/nuPLN/nuPLN.tex:L55-L79] [docs/nuPLN/nuPLN.tex:L413-L418]

## 3. Alternative truth-value types (`Bl`, `Pr`, `STV`, `ETV`, `TTV`)

C-3-1: `TruthValue.metta` defines exactly three core constructors: `Bl : Bool -> TruthValue`, `Pr : Number -> TruthValue`, and `STV : Number Number -> TruthValue`. The comments label `Pr` as "mere probability" and `STV` as the second-order probability constructor. [experimental/early-pln/common/truthvalue/TruthValue.metta:L16-L26]

C-3-2: `Bl` and `Pr` are not the missing richer TV hierarchy from the book. Their `count` is forced to `max-count = 1e9` and their `confidence` is forced to `1.0`, which makes them degenerate "full-certainty" wrappers rather than implementations of `IndefiniteTruthValue` or `DistributionalTruthValue`. [experimental/early-pln/common/truthvalue/TruthValue.metta:L43-L45] [experimental/early-pln/common/truthvalue/TruthValue.metta:L97-L112]

C-3-3: Only `STV` participates in real evidence accumulation. `inc-neg-count` and `inc-pos-count` are defined solely on `STV`, converting confidence to count, changing the positive/total counts, and returning a new `STV`. That means the repo's real inductive updating machinery still lives in the classic `(strength, confidence)` pair, not in `Bl` or `Pr`. [experimental/early-pln/common/truthvalue/TruthValue.metta:L51-L83]

C-3-4: `ETV` is a wrapper around an existing `TruthValue`, not a new truth-value family. `EvidentialTruthValue.metta` defines `ETV (OrderedSet a) TruthValue`, and all accessors (`mode`, `count`, `confidence`, `mean`) simply delegate to the wrapped TV while carrying explicit evidence alongside it. [experimental/early-pln/common/truthvalue/EvidentialTruthValue.metta:L17-L27] [experimental/early-pln/common/truthvalue/EvidentialTruthValue.metta:L29-L61]

C-3-5: The temporal layer is also only a wrapper at this repo snapshot. `TemporalTruthValue.metta` defines `TTV` as a map from time to `TruthValue` and `TETV` as a map from time to `EvidentialTruthValue`, but it contains no temporal inference rules. So the repo names temporal truth values without recovering the temporal PLN rule families themselves. [experimental/early-pln/common/truthvalue/TemporalTruthValue.metta:L1-L6] [experimental/early-pln/common/truthvalue/TemporalTruthValue.metta:L14-L26]

## 4. Missing-family recovery check

C-4-1: The actual implemented rule surface under `experimental/early-pln/pln/` contains only three rule families: Deduction, Modus Ponens, and Implication Direct Introduction. There are no checked-in rule files for quantifiers, temporal/causal operators, concept geometry, multideduction, BRGI, OBI, attention/importance, FactorGraph, MORK, or context links. [experimental/early-pln/pln/README.md:L1-L8] [experimental/early-pln/pln/*]

C-4-2: The repo does not recover the book's intensional/extensional conversion machinery as code. The only explicit treatment of that topic is the manuscript argument that the split should collapse into a single extensional/predicate-side formalization via isomorphism. That is a methodological pivot, not an implemented conversion rule family. [docs/nuPLN/nuPLN.tex:L254-L315]

C-4-3: The repo does not recover temporal/causal or distributional families locally; it only points to them externally. The next-token-prediction use case explicitly points readers to the external Temporal PLN paper for predictive implication / sequential conjunction and to an old AtomSpace issue for category/generalized distributional truth values. [docs/use-cases/next-token-prediction/next-token-prediction.md:L74-L100] [docs/use-cases/next-token-prediction/next-token-prediction.md:L177-L210]

C-4-4: The conjunction-introduction experiments introduce the idea of a future `UTV` (Universal Truth Value) as the ideal integration object for confidence calculations, but `UTV` is only a research intention here. There is no checked-in `UTV` type or rule implementation. [experimental/truth-value-formula/conjunction-introduction/README.md:L8-L20]

C-4-5: The one genuinely new rule surface relative to Sources 2-3 is Implication Direct Introduction, including a base case and a recursive evidence-accumulation rule. That is new, but it does not broaden the repo toward quantifiers/temporal/intensional/distributional/FactorGraph families. [experimental/early-pln/common/formula/ImplicationDirectIntroductionFormula.metta:L6-L19] [experimental/early-pln/pln/dependent-types/ImplicationDirectIntroductionDTL.metta:L80-L144]

## 5. Confidence-formula continuity vs. the `pln0.9` pivot

C-5-1: `pln-experimental2` predates the Source 3 multiplicative-confidence pivot. Its implemented confidence formulas are still min/count based: deduction returns `min($Pc, $Qc, $Rc, $PQc, $QRc)`, Modus Ponens returns `min($ABc, $notABc, $Ac)`, and Implication Direct Introduction updates confidence only through `count->confidence` / `inc-pos-count` / `inc-neg-count`. [experimental/early-pln/common/formula/DeductionFormula.metta:L38-L54] [experimental/early-pln/common/formula/ModusPonensFormula.metta:L7-L14] [experimental/early-pln/common/formula/ImplicationDirectIntroductionFormula.metta:L8-L19]

C-5-2: Modus Ponens does contain the same `0.2` anomaly seen elsewhere, but here it is explicit research debt rather than a silent carryover. `ModusPonensFormula.metta` hardcodes `$notABs 0.2` as a "temporary hack", and `ModusPonensDTL.metta` repeats that this should ideally be replaced by a precise Modus Ponens requiring an explicit `P(B|Not A)` premise. [experimental/early-pln/common/formula/ModusPonensFormula.metta:L7-L14] [experimental/early-pln/pln/dependent-types/ModusPonensDTL.metta:L12-L27]

C-5-3: I did not find any `Truth_or`-style heuristic, multiplicative-with-strength confidence formula, or generalized pln0.9-style confidence rewrite in this repo's implemented rules. The experimental confidence work here is happening in the Maxima derivations and nuPLN manuscript, not in the current executable rule formulas. [experimental/early-pln/common/formula/*.metta] [experimental/truth-value-formula/conjunction-introduction/README.md:L8-L20]

C-5-4: This makes Source 4 semantically orthogonal to Source 3. `PLN/` changed executable confidence propagation across its rule core; `pln-experimental2` instead explores successor semantics on paper and in Maxima, while its checked-in MeTTa rules remain closer to the older/min-based regime. ## 6. Maxima symbolic derivations

C-6-1: The Maxima work is focused on conjunction introduction, not on the currently implemented Deduction/MP/IDI trio. The README frames the goal as deriving confidence calculations that are efficient yet justified by probability theory, ideally as approximations to an uncomputable universal integration, starting with conjunction introduction "for its simplicity." [experimental/truth-value-formula/conjunction-introduction/README.md:L6-L20]

C-6-2: The first Maxima result argues that when `A` and `B` are uniformly randomly distributed and the universe size tends to infinity, the distrolet for `pAB` tends to a Dirac delta with mean `pA*pB`, i.e. the independence assumption becomes justified in that limit. The README gives the closed-form `Pr(pAB = x)` expression derived using Stirling approximation. [experimental/truth-value-formula/conjunction-introduction/README.md:L60-L95] [experimental/truth-value-formula/conjunction-introduction/README.md:L111-L123]

C-6-3: The same research track then introduces extra dependency premises (`A -> B` and later `A <-> B`) and studies how these change the concluding second-order distribution. The README concludes that structure itself can provide evidence, and the `conjunction-xp.mac` script contains the supporting symbolic/probabilistic machinery, including unfinished scaled-beta fitting functions left as `NEXT`. [experimental/truth-value-formula/conjunction-introduction/README.md:L132-L183] [experimental/truth-value-formula/conjunction-introduction/README.md:L349-L358] [experimental/truth-value-formula/conjunction-introduction/conjunction-xp.mac:L170-L180] [experimental/truth-value-formula/conjunction-introduction/conjunction-xp.mac:L319-L321]

C-6-4: These derivations do not validate or invalidate the Source 3 `pln0.9` pivot directly, because the symbolic target here is conjunction introduction, a rule family that is not ported into the repo's MeTTa PLN encodings. They are best read as future-facing semantic experiments rather than as support for the current executable rule formulas. [experimental/truth-value-formula/conjunction-introduction/README.md:L18-L20] [experimental/truth-value-formula/conjunction-introduction/conjunction-xp.mac:L247-L259]
