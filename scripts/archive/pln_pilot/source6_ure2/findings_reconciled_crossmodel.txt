## 1. Headline finding: URE was a research **extension** of the 2008 book, abandoned wholesale by the MeTTa line

**[X3]** This is the cluster-level reframing of the entire PLN pilot:

- **Source 1 (book 2008)** describes BIT (And-Or backward inference tree) + multi-armed bandit + SoftMax rule selection.
- **URE (this source)** implements the BIT, then **goes beyond** the book with: Bayesian Thompson Sampling (proper posterior, not heuristic SoftMax), BetaDistribution per-rule TV→posterior conversion, MixtureModel for combining multiple active control rules in Universal-Operator-Induction style, complexity penalty + Kolmogorov heuristic `K(D) = |D|^(1-c)`, control-rule meta-layer (rules about rules) stored as Atomese in a separate atomspace, and probabilistic BIT pruning via `remove_unlikely_expandable_andbit`.
- **MeTTa-era PLN line (Sources 2-5)** dropped all of it. Replaced with a simple confidence-priority queue (`PLN.Derive` PQ-based forward chaining; `MaxSteps`, `TaskQueueSize`, `BeliefQueueSize` are the only knobs).

**The MeTTa simplification is far more aggressive than the book itself.** URE was not just an implementation of the 2008 book — it was a research-grade extension of it, and the MeTTa abandonment lost both the extensions AND much of the book-level original. This is the most striking finding of the entire PLN cluster pilot to date.

## 2. The Bayesian inference-control architecture (X3 verbatim verification)

URE's full control stack:

| Component | File | Function |
|---|---|---|
| **`Rule`** | `opencog/ure/Rule.h:157` | A `BindLink` over `<vars><AndLink<premises>><conclusion>` or with an `ExecutionOutputLink` formula body. Generic Atomese, not specialized C++ rule classes. |
| **`RuleSet`** | `Rule.h:55-112` | Sorted vector with `expand_meta_rules()` for rule-producing rules. |
| **`UREConfig`** | `UREConfig.h:46` | **Reads config FROM AtomSpace** (data-driven, not static files) — `URE:maximum-iterations`, `URE:complexity-penalty`, `URE:expansion-pool-size`, etc. |
| **`BetaDistribution`** | `BetaDistribution.h:40` | TV → Beta posterior via `pos_count = mean × count`. Boost-backed. CDF/PDF helpers. |
| **`ThompsonSampling`** | `ThompsonSampling.h:41` | Discretized arm selector: `Pi = ∫ pdf_i(x) Π_{j≠i} cdf_j(x) dx / nt`. Doc: "See Section Inference Rule Selection in PLN inference-control-learning README" (out-of-tree). |
| **`ActionSelection`** | `ActionSelection.h:50` | Handle→TV map wrapped around `ThompsonSampling`. Returns `HandleCounter distribution()` or single `Handle operator()()`. |
| **`MixtureModel`** | `MixtureModel.h:39` | Combines partial-models via Universal Operator Induction-style mixture. `prior(length) = exp(-cpx_penalty × length)` with Kolmogorov heuristic for un-modeled data. Compressiveness ∈ [0,1] interpolates "no compression" ↔ "perfect compression". |
| **`BIT` / `AndBIT` / `BITNode`** | `backwardchainer/BIT.h:48-435` | Full And-Or-tree backward inference. Each `AndBIT` is a forward-chaining-strategy candidate; `BITNode` is an or-leaf with rule choices. Includes `fcs_to_ascii_art()` for visualization. |
| **`ControlPolicy`** | `backwardchainer/ControlPolicy.h:48` | Orchestrator: takes config + BIT + target + optional `_control_as` (atomspace of control rules). Selects rules via Thompson posterior or MixtureModel of control-rule predictions. |
| **`BackwardChainer`** | `backwardchainer/BackwardChainer.cc` | Three-phase loop: `expand_bit()` → `fulfill_bit()` → `reduce_bit()`. Probabilistic survival pruning when BIT exceeds `max_bit_size`. |
| **`ForwardChainer`** | `forwardchainer/ForwardChainer.h:51` | Three modes: `do_steps_singlethread`, `do_steps_multithread`, `do_steps_srpi` (Source-Rule Producer). Source-selection enum: `TV_FITNESS / STI / UNIFORM` (STI = ECAN attention). |
| **`SourceRuleSet`** | `forwardchainer/SourceRuleSet.h:83` | "Expansion Pool" — pairs of (Source, Rule) weighted by 2nd-order TV. Holds its own `ThompsonSampling`. |

**[X3 with Claude verbatim transcription]** The full algorithmic story (Claude C-10-9): *"At each `do_step`, BackwardChainer chooses an AndBIT by `discrete_distribution` weighted by `andbit_fitness × exp(-cpx_penalty × andbit.complexity)`. Within that AndBIT, picks a leaf via `BITNode::operator()()` (anti-fitness weighted: pick the unfit leaf to grow). For that leaf, ControlPolicy considers all valid rules and weights them by `expansion_success_tvs` — computed by either `_default_tvs` fallback or `MixtureModel({active_control_rules}, cpx_penalty, compressiveness)`. The chosen weights become a `BetaDistribution`-typed `HandleTVMap`, fed into `ActionSelection` → `ThompsonSampling::operator()` → returns the chosen rule. Rule expands the AndBIT into a successor (probability `prob` becomes part of successor's `complexity`). BIT is pruned by `discrete_distribution` over `(1-p)^remaining_iterations`."*

## 4. Rule families present in URE that were lost downstream

URE has a more substantive rule library than the entire MeTTa-era PLN cluster has produced. Per `tests/ure/rules/` (16 .scm files):

**Implemented in URE, not in MeTTa-era PLN:**
- `intensional-inheritance-direct-introduction.scm` — **the exact intensional inheritance machinery from book Ch. 5.10**, using `AttractionLink` + pattern-of(X,A) = s(X) × (P(X|A) − P(X|¬A))_+. **Major finding: the book's intensional-extensional treatment lives here in C++ but never made it into MeTTa.** [X3]
- `fuzzy-conjunction-introduction-rule.scm` — fuzzy logic conjunction with TV. Absent in MeTTa.
- `implication-instantiation-rule.scm` + `conditional-full-instantiation-meta-rule.scm` + `conditional-partial-instantiation-meta-rule.scm` — **ImplicationScope quantifier instantiation via meta-rules**. The "quantifier rules" we've been tracking as missing are actually here, just expressed via meta-rule expansion rather than dedicated Atomese types. [X3]
- `implication-scope-direct-evaluation-rule.scm` — direct TV evaluation from data.
- `closed-lambda-introduction-rule.scm` + `implication-and-lambda-factorization-rule.scm` — higher-order (lambda) machinery.
- `evidence-based-conditional-direct-evaluation.scm` — **STUBBED, work paused**: comment says *"TODO: resume once GlobNode is supported"* (variadic match, never finished in AtomSpace). Permanent abandonment signal.

**Absent in URE too** (still missing, all three sources verified):
- Temporal/causal (PredictiveImplication, BeforeLink, AfterLink) — book Ch. 14, never in C++.
- Distributional TV beyond Beta (no continuous-distribution TV class).
- FactorGraph / MORK / BRGI / OBI.
- Explicit ForAll/Exists symbols (handled via ImplicationScope+meta-rules).
- Causal calculus.
- ECAN integration beyond the `STI` source-selection-mode hook (the actual attention math lives in the separate `opencog/attention` repo).
