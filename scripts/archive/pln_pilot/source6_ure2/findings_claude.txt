## §2 — Inference control: MAB / ControlPolicy / ActionSelection (PRIORITY OUTPUT)

C-2-1. **YES — URE actually implements a Bayesian Thompson-Sampling MAB control system.** This is fundamentally absent from MeTTa-era PLN3/PLN/.

C-2-2. The architecture:
  - `BetaDistribution` (per-rule): converts a `TruthValuePtr` (mean/confidence pair) into a Beta posterior over success-probability.
  - `ThompsonSampling`: takes a `TruthValueSeq` and produces `distribution()` (probability that each arm is the best) or samples directly via `operator()()`.
  - `ActionSelection`: maps Handle→TV via `HandleTVMap`, wraps `ThompsonSampling`, returns `HandleCounter distribution()` or single `Handle operator()()`.
  - `ControlPolicy` (BC-side): orchestrates rule selection given an AndBIT + BITNode + control rules.
  - `MixtureModel`: combines multiple control rules into one TV when several apply.

C-2-3. `ThompsonSampling.h:62-67` (verbatim): "Pi = I_0^1 pdfi(x) Prod_j!=i cdfj(x) dx / nt" — the Bayesian "probability arm i is best" formula. Doc reference: "See Section Inference Rule Selection in the README.md of the pln inference-control-learning". This is **not** SoftMax; it's the proper Thompson posterior.

C-2-4. `ThompsonSampling.cc:65-86` (sampling implementation): for size N>1, sample one Beta first-order probability per arm, take argmax (with random tie-break among maxima) — this is textbook Thompson sampling.

C-2-5. `ThompsonSampling.cc:40-63` (distribution implementation): computes per-arm cdf with `_bins=100`, then for each arm i computes `Pi = sum over bins of pdf_i(x) * Prod_{j!=i} cdf_j(x)` (right-end Riemann), then normalizes.

C-2-6. `BetaDistribution.h:47-50` constructors: `BetaDistribution(const TruthValuePtr& tv, double prior_alpha=1.0, double prior_beta=1.0);` and `BetaDistribution(double pos_count, double count, double prior_alpha=1.0, double prior_beta=1.0);` — uses Bayes-Laplace prior by default.

C-2-7. `BetaDistribution.cc:33` comment: `"// TODO should be replaced by tv->get_mode() once implemented"` — confirms the alpha/beta is derived from TV mean/count via the standard pos_count = mean*count formula.

C-2-8. `ActionSelection.h:60-76`: doc says "The distribution attempts to reflect the optimal balance between exploration and exploitation (Thompson sampling)". ActionSelection itself is essentially a Handle-keyed wrapper around ThompsonSampling.

C-2-9. `MixtureModel.h:39-107`: combines multiple "active partial models" (control rules whose preconditions match) into one TV via "Sum_i Pi * pdf_i * Beta(alpha_i, beta_i)" up to normalization (MixtureModel.h:80). Pi = `prior(length + K(D))` where `K(D) = |D|^(1-c)` (Kolmogorov heuristic, MixtureModel.h:88-95). `prior(length) = exp(-cpx_penalty * length)` (MixtureModel.h:148-160). This is **Universal Operator Induction** style mixture (term used verbatim, MixtureModel.h:78). The compressiveness parameter c ∈ [0,1] interpolates between "no compression" and "perfect compression" (MixtureModel.h:138-144).

C-2-10. `backwardchainer/ControlPolicy.h:48`: `class ControlPolicy` is the orchestrator. CTor: `ControlPolicy(const UREConfig& ure_config, const BIT& bit, const Handle& target, AtomSpace* control_as=nullptr);`. Public surface:
  - `RuleSet rules;` (the inference rule pool)
  - `RuleSelection select_rule(AndBIT& andbit, BITNode& bitleaf);` (where `RuleSelection = pair<RuleTypedSubstitutionPair, double>` — selected rule + 2nd-order TV mean for AndBIT complexity calc; ControlPolicy.h:46).

C-2-11. `ControlPolicy.h:97`: `HandleTVMap _default_tvs;` — fallback per-rule TV used when no control-rule applies. `ControlPolicy.h:108`: `AtomSpace* _control_as;` — atomspace holding **inference control rules** (a meta-level: rules about which rules to use). `ControlPolicy.h:116`: `std::map<Handle, HandleSet> _expansion_control_rules;`.

C-2-12. `ControlPolicy.h:122-148`: control flow:
  - `get_valid_rules(andbit, bitleaf)` → `RuleTypedSubstitutionMap` (which rules can possibly infer the target).
  - `expansion_success_tvs(andbit, bitleaf, rules)` → `HandleTVMap` (per-rule probability that expanding succeeds, derived from control rules in `_control_as`).
  - `rule_weights(success_tvs, rules)` → `vector<double>` for weighted random selection.
  - `default_alias_weights(rules)` → fallback when no control rule applies.

C-2-13. `ControlPolicy.h:208-228` (verbatim doc snippet): control rules have the schema
```
ImplicationScope <TV>
  <vardecl>
  And
    Evaluation Predicate "URE:BC:preproof-of" List <A> <T>
    Execution Schema "URE:BC:expand" List <A> <L> <inf_rule> <B>
    <pattern-1> ... <pattern-n>
  Evaluation Predicate "preproof-of" List <B> <T>
```
This is the **ControlPolicy.h:255-296** mechanism: control rules predict whether expanding andbit A by inference rule `inf_rule` yields a successor andbit B that is still a "preproof" of target T. Predicate name `"URE:BC:preproof-of"` (ControlPolicy.h:56).

C-2-14. `BackwardChainer::expand_bit(AndBIT& andbit)` (BackwardChainer.cc:183-238): uses `_control.select_rule(andbit, *bitleaf)` to pick a rule, then expands. AndBIT expansion includes a probability `prob` flowing into the AndBIT complexity (BackwardChainer.cc:227-230).

C-2-15. AndBIT selection for expansion (BackwardChainer.cc:297-323): `expansion_andbit_weights()` builds weights from `_andbit_fitness(andbit) * complexity_factor(andbit)` where `complexity_factor = exp(-complexity_penalty * andbit.complexity)` (BackwardChainer.cc:387-397). Then `std::discrete_distribution<size_t>` sampling. **This is BIT-level priority sampling** (not the per-rule Thompson sampling — that is at the rule-selection step inside ControlPolicy).

C-2-16. AndBIT pruning when BIT exceeds `max_bit_size` (BackwardChainer.cc:330-385): `remove_unlikely_expandable_andbit()` computes `nep = (1-p)^(remaining_iterations)` per AndBIT and removes via `discrete_distribution` weighted by nep — i.e. it preferentially drops AndBITs that have low chance of being picked in any of the remaining iterations. **This is a probabilistic survival mechanism specific to URE.**

C-2-17. ForwardChainer rule selection (ForwardChainer.cc:43-44 includes; ForwardChainer.h:202-207): `select_rule()` overloads. The forward-chainer uses `SourceRuleSet::thompson_select()` (SourceRuleSet.h:100, .h:134 holds `ThompsonSampling _thompson_smp;`) for Source-Rule pair selection in the `srpi` (Source-Rule-Producer-Implementation) pathway. Source selection mode is configurable: `TV_FITNESS / STI / UNIFORM` (ForwardChainer.h:40-42). The STI option suggests originally-planned ECAN attention integration.

**Delta vs MeTTa-era PLN line (Sources 2-5):**
  - PLN3/PLN/'s Deriver.metta uses a simple **confidence-priority queue** (`TaskQueueSize=20→10`, `BeliefQueueSize=200→100` from Source 3 reconciliation). No Beta posteriors. No Thompson sampling. No control-rule meta-layer. No Mixture Model. No complexity-weighted survival. → **The full Bayesian inference-control machinery is unique to URE; the MeTTa line abandoned it entirely.**

## §4 — `forwardchainer/` and `backwardchainer/` deep dive

C-4-1. `backwardchainer/` files: `BIT.{h,cc}`, `BackwardChainer.{h,cc}`, `ControlPolicy.{h,cc}`, `Fitness.{h,cc}`, `TraceRecorder.{h,cc}`. 5 .h + 5 .cc + 1 CMakeLists.

C-4-2. **YES — the Backward Inference Tree (BIT) is fully implemented** (BIT.h:435 `class BIT { AtomSpace bit_as; AndBITs andbits; ... }`). And-Or-tree structure: `BITNode` (BIT.h:48) with `RuleTypedSubstitutionMap rules` (or-children = which rule), `AndBIT` (BIT.h:81) holds `Handle fcs` (a forward-chaining-strategy hypergraph) + `HandleBITNodeMap leaf2bitnode` (and-children = unproven sub-targets).

C-4-3. The BIT data structure in URE corresponds **directly** to the 2008 PLN book §4.2 description. The And-Or-tree-with-FCS (forward-chaining-strategy) representation is the URE implementation of book-style backward inference.

C-4-4. `BackwardChainer::do_step()` (BackwardChainer.cc:102-112):
```
expand_bit();   // grow BIT by one rule application
fulfill_bit();  // try to actually run the FCS over the AtomSpace
reduce_bit();   // prune oversized BIT
```

C-4-5. `BackwardChainer::expand_bit()` (BackwardChainer.cc:162-181):
  - `expand_meta_rules()` (BackwardChainer.cc:145-160) — per iteration, runs meta-rules forward to generate new instance rules and adds them to `_rules`. **Comment** at BackwardChainer.cc:147: "This is kinda of hack before meta rules are fully supported by the Rule class."
  - If BIT empty: `_bit.init()`. Else: `select_expansion_andbit()` (BackwardChainer.cc:305-323) → weighted random andbit pick → `expand_bit(andbit)`.

C-4-6. `BackwardChainer::expand_bit(AndBIT& andbit)` (BackwardChainer.cc:183-238): selects leaf via `andbit.select_leaf()` (fitness-weighted; BIT.h:139-145), selects rule via `_control.select_rule(andbit, *bitleaf)` (the ControlPolicy / Thompson Sampling path), and then `_bit.expand(andbit, *bitleaf, rtsp, prob)`. Records expansion in trace atomspace if successful.

C-4-7. `BITNode::operator()()` (BIT.cc:59-68) — the "probability of usefulness of expanding this node":
```c++
return (exhausted ? 0.0 : 1.0) *
    (fitness.upper - fitness(*this)) / (fitness.upper - fitness.lower);
```
Anti-proportional to fitness (higher fitness → less to gain by expanding). This is **the rule-input-filter / output-based-input-filter analog** from book §4.2 — implemented as a single fitness-driven weight.

C-4-8. `BITNodeFitness` (Fitness.h:38-57): only one mode — `MaximizeConfidence`. Holds `std::function<double(const BITNode&)> function; double lower; double upper;`. Co-domain bounds let the BITNode formula above remap to [0,1].

C-4-9. `AndBITFitness` (Fitness.h:62-94): two modes — `Uniform` (always 1.0) and `Trace` (1.0 iff FCS is in pre-recorded trace, 0.0 otherwise; "handy for running inferences without the computational cost of searching the trace"). The trace mode is for **inference replay** (used by `inference-control-learning` per the doc references).

C-4-10. `forwardchainer/` files: `ForwardChainer.{h,cc}`, `FCStat.{h,cc}`, `SourceSet.{h,cc}`, `SourceRuleSet.{h,cc}`, `README.md`.

C-4-11. `ForwardChainer::do_chain()` (ForwardChainer.cc:120-158) decides between three execution modes:
  - empty-source: `apply_all_rules()` (legacy Relex2Logic path).
  - `_srpi=true` (default; "likely temporarily" comment at ForwardChainer.h:255-256): Source-Rule Producer Implementation.
  - `_srpi=false`: `do_steps_singlethread()` or `do_steps_multithread()` (depends on `_config.get_jobs()`).

C-4-12. `ForwardChainer::do_step_srpi()` is the modern path: populate `_source_rule_set` (Expansion Pool) with weighted (Source*, Rule*) pairs, then `thompson_select()` from it.

C-10-6. **AtomSpace coupling**: URE is fundamentally tied to the C++ AtomSpace (`#include <opencog/atomspace/AtomSpace.h>` is everywhere). The MeTTa interpreter uses its own atom representation. → The migration cost from URE to MeTTa equivalents was **complete reimplementation in a different host language**, not porting. C-10-7. **The "control-rule" abstraction is unique to URE** (ControlPolicy.h:208-296). Control rules are atoms in `_control_as` that predict expansion success. The pattern is: "if andbit A is preproof of T and we expand it with rule R from leaf L producing andbit B and these patterns hold, then B is also preproof of T with conditional probability TV." This kind of meta-reasoning machinery is **completely absent in MeTTa-era PLN**, where rule selection is fixed at config time.

C-10-8. **Multi-armed bandit framing**: URE explicitly treats rule selection as a contextual MAB (BIT-leaf is context, rule is arm, success-probability TV is reward distribution). Books and PLN papers from 2008-2018 era discussed this; URE is the only known implementation. The MeTTa line dropped MAB framing entirely.

C-10-9. **The complete URE control architecture, in one paragraph**: At each `do_step`, the BackwardChainer chooses an AndBIT (proof candidate) by `discrete_distribution` weighted by `andbit_fitness * exp(-cpx_penalty * andbit.complexity)`. Within that AndBIT, it picks a leaf via `BITNode::operator()()` (anti-fitness weighted: pick the unfit leaf to grow). For that leaf, ControlPolicy considers all valid rules (those whose conclusion can unify with the leaf) and weights them by `expansion_success_tvs` — which is computed by either: (a) the `_default_tvs` fallback, or (b) `MixtureModel({active_control_rules}, cpx_penalty, compressiveness)` if any control rule's pattern matches. The chosen weights become a `BetaDistribution`-typed `HandleTVMap`, fed into `ActionSelection` → `ThompsonSampling::operator()` → returns the chosen rule. The rule expands the AndBIT into a successor AndBIT (probability `prob` becomes part of the successor's `complexity`). The BIT is pruned by `discrete_distribution` over `(1-p)^remaining_iterations` to keep `|BIT| ≤ max_bit_size`. Compared to the book's BIT+SoftMax description (Source 1 §4.2-4.3), URE replaces SoftMax with Thompson Sampling (which is more principled Bayesian) and adds a complexity-penalty layer plus a control-rule meta-layer.

C-10-10. **Observed wiki-outreach implication** (per brief): For URE-related cards, Linas Vepstas is the authoritative expert. For the inference-control / Bayesian / MAB material specifically, Nil Geisweiller is THE primary author (every relevant file's copyright says "Author: Nil Geisweiller").

C-10-11. **Provenance-claim cross-link**: Source 1 (book 2008) says "BIT + MAB". URE (this source) says "BIT + Bayesian Thompson + MixtureModel + control rules" (more sophisticated than the book's SoftMax). MeTTa line (Sources 2-3-4-5) says "priority queue, no BIT". → **The MeTTa simplification is far more aggressive than the book itself**. URE was not just an implementation of the book — it was a research extension of it, and the abandonment lost both the extensions AND the book-level original.
