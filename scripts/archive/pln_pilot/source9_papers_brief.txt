## Standing corrections to carry into the bundle

(Sourced from Sources 2-8 cross-model reconciliations. Treat these as authoritative; if a paper claim contradicts them, the *paper* is wrong/aspirational, not the standing correction.)

1. **MORK is a substrate only.** No checked-in FactorGraph implementation. Zero `FactorGraph`/`MessagePass`/`BeliefPropagation` matches in MORK source. Source 8 reconciliation confirms.
2. **`.mm2` is MORK's executable rewrite-program format**, NOT Metamath. The file extension is naming convention; the format is MeTTa S-expressions with `(exec <loc> <pat> <tpl>)` rewrite syntax. Source 8 §3.
3. **`chaining` shows only partial migration of URE control ideas.** ~5 of 16 URE features survive at toy scale. BIT, MixtureModel, full Thompson Sampling all absent. Source 7 reconciliation §1-2.
4. **The current production PLN3/PLN/ line has no serious inference-control layer** — only a confidence-priority forward-chaining queue. Source 3 reconciliation. Sources 6-7 confirmed the URE/chaining migration didn't close this gap.
5. **pln0.9 introduced a real confidence-model pivot** — 7 of 11 rules changed from min-based to multiplicative confidence with strength-entanglement, plus introduction of `Truth_or` heuristic. Source 3 reconciliation §2.
6. **The "0.2 MP hack" provenance is leungmanhin (PR #23 hyperon-pln, 2023-07)**, NOT later authors. The 0.2 lives in `Truth_SymmetricModusPonens`; the 0.02 `Truth_ModusPonens` is a separate post-2023 book-faithful addition. Source 5 reconciliation §1.
7. **`PLN AND Confidence Framework.md.pdf` was referenced as "Ben's document"** in pln-experimental2 (Source 4). Provenance is unverified beyond the local-PDF reference. **Treat the wiki's `RawData+Publications+PLN AND Confidence Framework.md` card as local/unpublished unless content + comparison to the Source 4 reference establishes it as a published document.**
8. **The dead-infrastructure pattern**: probabilistic primitives are imported as affordances across the cluster but never integrated into reasoning code. Examples: chaining's unused `BetaDistribution` (`experimental/common/truthvalue/`), MORK's unused `distr-combinators` Cargo dep, MORK's `experiments/eval/` and `eval-ffi/` work loosely tied to TVs but not for Bayesian inference. **When a paper proposes "use Beta posteriors here" — verify against this pattern. The math may exist but isn't wired in.**
9. **MORK is in a distinct ecosystem from the PLN cluster.** Vandervorst / Clarke / Peterson lead MORK; Geisweiller has 1 commit. The "PLN-on-MORK" connection is **inter-ecosystem**, socially as well as technically unbuilt.
10. **chaining's `experimental/pln/` uses pre-pln0.9 (min-based) confidence**, NOT the production pln0.9 multiplicative pivot. Two PLN math lineages coexist in the cluster.

## Special handling for the FactorGraph MORK iterative version stream

The 5 FactorGraph cards (`v1`, `v2`, `v3.1`, `v3.2`, plus the unversioned base) are an **iterative paper stream**, not 5 independent papers. Treat as one paper across versions. After per-version extraction, produce a **version-to-version delta table**:

```
| Claim / proposal | v1 | v2 | v3.1 | v3.2 |
|---|---|---|---|---|
| <claim 1> | introduced | revised | dropped | reintroduced |
| <claim 2> | absent | introduced | persisted | persisted |
| ...
```

If a claim first appears in one version and disappears later, note that explicitly. The version stream may show:
- Refinements (v1 has an idea, v2 sharpens it).
- Abandonment (v1 has an idea, v3.x drops it — possibly indicating implementation difficulty).
- New introductions (v2 adds a concept absent from v1).
- Persistence (claim stays through all versions — likely the most robust proposal).
