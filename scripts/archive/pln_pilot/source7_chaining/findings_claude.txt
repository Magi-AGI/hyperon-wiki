## §1 — Architecture map (`dtl/` vs `experimental/`)

**C-1-1 [dtl/ scope]** `dtl/` is the closest thing to a reusable module surface. Its tree is exactly: `dtl/utils.metta`, `dtl/forward/{curried,no-curry}.metta`, `dtl/backward/{curried,no-curry}.metta`, `dtl/tests/{test-utils, test-forward-chaining-{curried,no-curry}, test-backward-chaining-{curried,no-curry}}.metta`. Five files of code + five tests. (`dtl/`)

**C-1-2 [dtl/utils.metta scope]** 38 lines total. Defines `add-atom-nodup`, `add-atoms-nodup`, and the `Nat` type (`Z`, `S`, `fromNumber`, `fromNat`). NO truth-value machinery, NO control-flow. This is the *complete* utility surface for `dtl/`. (`dtl/utils.metta:L1-L38`)

**C-1-3 [DTL backward no-curry signature]** `(: syn (-> $a Nat $b $b))` — knowledge-base, max-depth, query, result. 6-line core: base case (`match $kb $query $query`) plus binary/ternary recursive steps. NO truth values, NO Bayesian sampling, NO BIT-style And-Or tree. This is plain dependently-typed proof synthesis, the same algorithm from Source 5's `metta/synthesis/Synthesize.metta`. (`dtl/backward/no-curry.metta:L21-L41`; cf. Source 5 `Synthesize.metta`.)

**C-1-4 [DTL backward curried adds isyn iterative wrapper]** `(: isyn (-> $a Nat Nat $b $b))` — adds an iteration count, calls `syn` then writes results back to `$kb` between iterations via `add-atoms-nodup`. *No control beyond max-depth/iterations.* (`dtl/backward/curried.metta:L33-L47`)

**C-1-5 [DTL forward fcc reuses bc internally]** Comment at `dtl/forward/curried.metta:L31-L34`: *"TODO: Import the curried backward chainer instead of repeating the definition here."* `bc` is duplicated inside `curried.metta:L42-L48`. Forward chaining "unfolds the rules backward" — the fc chainer calls bc with a rule abstraction as the query. (`dtl/forward/curried.metta:L1-L92`)

**C-1-6 [DTL no-curry forward = 21 lines]** Forward chainer in `dtl/forward/no-curry.metta`: 2 recursive cases over binary rules, no max-depth control, no truth values. (`dtl/forward/no-curry.metta:L1-L21`)

**C-1-7 [experimental/ tree size]** 22 numbered subdirectories (excluding `ArgSetChainer/` submodule + 1 README): `backward-chaining/`, `common/`, `converters/`, `curried-chaining/`, `evolutionary-programming/`, `forward-chaining/`, `implicit-argument/`, `inference-control/`, `iterative-chaining/`, `lambda-abstraction/`, `metamath/`, `modal-logic/`, `no-depth/`, `pln/`, `pln-inf-ctl/`, `polyward-chaining/`, `prob-chaining/`, `program-verification/`, `proof-tree/`, `subtyping/`, `sumo/`, `synthesis/`. (Listing under `experimental/`)

**C-1-8 [experimental/common/ contents]** `Maybe.metta`, `In.metta`, `OrderedSet.metta`, `Record.metta`, `EqualityType.metta`, `Num.metta`, `List.metta`, `BelieveMe.metta` plus `truthvalue/` and `formula/` subdirs. The `BelieveMe.metta` file is the same anti-pattern from Source 5. (`experimental/common/`)

**C-1-9 [experimental/common/formula/ identifies as Source-5-derived]** Files: `DeductionFormula.metta`, `DeductionFormulaTest.metta`, `ImplicationDirectIntroductionFormula.metta`, `ModusPonensFormula.metta`. The `DeductionFormula.metta` opens with the same consistency-condition machinery (`smallest-intersection-probability`, `largest-intersection-probability`, `conditional-probability-consistency`) verbatim from Source 5's seed (and forwarded into PLN3 / PLN/). (`experimental/common/formula/DeductionFormula.metta:L1-L54`)

**C-1-10 [experimental/common/truthvalue/ has full Beta-distribution machinery]** `TruthValue.metta` defines `Bl`, `Pr`, `STV`, plus prior-alpha/prior-beta = 0.5 (Jeffreys), max-count = 1e9, lookahead = 1.0, count↔confidence conversions, `inc-pos-count`/`inc-neg-count`, `mode`, `count`, `confidence`, `pos-count`, `neg-count`, **`post-alpha`** (= `prior-alpha + pos-count`), **`post-beta`** (= `prior-beta + neg-count`), **`mean`** (= `α/(α+β)` of posterior). `EvidentialTruthValue.metta` adds `ETV` constructor with same posterior alpha/beta wrappers. *This is the most complete Beta-distribution-aware TV machinery in the entire MeTTa-era PLN cluster — but per §2 it is NOT used by the pln-inf-ctl experiment.* (`experimental/common/truthvalue/TruthValue.metta:L1-L143`; `EvidentialTruthValue.metta:L1-L62`)

DTL's `syn` is a *direct recursive descent* with no shared expansion-pool, no AndBIT structure, no fitness-weighted choice. **Not the same algorithm.** Simpler.

**C-1-12 [DTL test scope]** Five tests: `test-utils.metta`, `test-{forward,backward}-chaining-{curried,no-curry}.metta`. None test inference control or truth-value reasoning. (`dtl/tests/`)

**C-1-13 [DTL imports]** `dtl/` core has zero `import!` of common formulas or truth values. It is a *typed-proof-search* module, intentionally orthogonal to PLN. (Inspection of `dtl/forward/*.metta`, `dtl/backward/*.metta`, `dtl/utils.metta`)

**C-2-17 [Verbatim transcription of `thompson-sample`]** Critical for the brief — full source:
```
;; Thompson sample from a simple truth value.  For now only confidence
;; of 0 or 1 are supported.  This assumption allows us to operate
;; Thompson sampling without requiring a Beta distribution.  Indeed if
;; the confidence is 0 then it amounts to a uniform sampling, assuming
;; a flat Bayesian prior that is.  And if the confidence is 1 then we
;; just return the strength.
(: thompson-sample (-> PLN.TruthValue Number))
(= (thompson-sample (PLN.MkSTV $s 0))
   (random-float &rng 0 1))
(= (thompson-sample (PLN.MkSTV $s 1))
   $s)
```
(`experimental/pln-inf-ctl/pln-inf-ctl.metta:L730-L742`)

**Comparison vs URE.** URE's `ThompsonSampling.h:41` (Source 6 §2) implements: discretized arm selector `Pi = ∫ pdf_i(x) Π_{j≠i} cdf_j(x) dx / nt` over Beta-posterior-typed arms, with full pdf/cdf integration. The chaining version handles only the two boundary cases (no information, perfect information) — explicitly avoiding the Beta integral. **The function name is migrated; the algorithm is not.** This is the cluster's clearest "name-preserved, content-degenerated" migration.

**C-2-18 [Per-component score]** Counting status codes from the table: **0 full**, **4 simp** (BetaDistribution, ThompsonSampling, ActionSelection, ControlPolicy), **1 simp** (control-rule meta-layer in separate atomspace), **11 drop**. Of 16 evaluated URE features, **5 partial migrations and 11 drops**. The Bayesian-mathematical core (Beta + Thompson over a Mixture + complexity-penalty) is **not** alive at usable form. The architectural skeleton (orchestrator-with-pluggable-control + control-rules-in-atomspace) is alive at one-pedagogical-experiment scale.

## §3 — DTL core (`dtl/`)

**C-3-1 [DTL is a typed-proof synthesizer, not a TV-aware reasoner]** `dtl/` files do not import any truth-value module. The chainer signatures `(-> $a Nat $b $b)` (kb, depth, query, result) carry no truth values, no consistency conditions. (`dtl/{forward,backward}/{curried,no-curry}.metta`)

**C-3-2 [DTL backward no-curry algorithm]** Three equations: base case `(syn $kb $_ (: $prf $ccln)) = (match $kb (: $prf $ccln) (: $prf $ccln))`, binary recursive `(syn $kb (S $k) ...) → let* on n-ary rule pattern with two recursive calls on premises`, ternary similar. Hand-coded for arity 2-3. **No And-Or-tree, no shared expansion pool — direct recursive descent with depth limit.** (`dtl/backward/no-curry.metta:L1-L21`)

**C-3-3 [DTL backward curried adds rule abstraction]** `(syn $kb (S $k) (: ($prfabs $prfarg) $ccln))` recurses on **proof abstraction** AND **proof argument** (Curry-Howard): `((: $prfabs (-> $prms $ccln)) (syn $kb $k (: $prfabs (-> $prms $ccln))))`. This is the Source 5 / Source 4 / Roman-Treutlein-style curried backward chainer, now in dtl. (`dtl/backward/curried.metta:L19-L22`)

**C-3-4 [vs URE BackwardChainer]** URE's `BackwardChainer` (Source 6 §2): three-phase `expand_bit() → fulfill_bit() → reduce_bit()`. DTL's `syn`: direct recursion. No equivalence. URE is BIT-style; DTL is PQ-less, BIT-less typed-proof-search.

**C-3-5 [vs Source 3 `lib_pln.metta` Deriver]** PLN/'s `Deriver` is bounded-priority-queue forward chaining (Source 3 reconciliation §2). DTL's forward chainer is *also not PQ-based* — it iterates over collapsed superposition results, then add-atoms-nodup back to space. Different style from both URE BIT and PLN/ PQ-Deriver. (`dtl/forward/curried.metta:L82-L91` `ifcc`)

**C-3-6 [DTL has no truth-value reasoning]** None of the DTL files import `common/truthvalue/`. **DTL is the proof-search layer; truth-value reasoning lives elsewhere.** This is a separation-of-concerns choice.

**C-3-7 [DTL is the production-ish core per repo author intent]** `AGENTS.md:L4-L8` (chaining repo): *"`dtl/` is the closest thing to a reusable module surface."* CLAUDE.md echoes this. Module-import naming `chaining:dtl:backward:curried` confirms it's the only module with formal registration.

Same four-encoding pattern from `patham9/hyperon-pln` Source 5 §2.1. Each subdir contains `Deduction*.metta` and `ImplicationDirectIntroduction*.metta` files. (Listing of `experimental/pln/`)

**C-4-3 [vs production PLN/]** Source 3's `PLN/lib_pln.metta` has the multiplicative-confidence pivot (Source 3 §2). **C-4-4 [PLN rule families: deduction + IDI only]** Same narrow rule set as Source 5: `Deduction`, `ImplicationDirectIntroduction`. ModusPonensFormula.metta also exists. No Induction, Abduction, Inversion, Revision in `experimental/pln/`. (Listing of `experimental/pln/dependent-types/`)

**C-4-5 [pln-inf-ctl defines its own PLN, different from experimental/pln/]** `pln-inf-ctl.metta:L608-L725` defines `PLN.TruthValue`, `PLN.MkSTV`, `PLN.Term`, `PLN.Statement`, `𝛩` (theory-proof-theorem ternary predicate), `≞` judgment connector, `PLN.reduce`, `pln-bc`, `EQI`, `CI`. **This is a new, simplified PLN dialect built specifically for the inference-control experiment, NOT importing `experimental/pln/` or `experimental/common/formula/`.** Its rule inventory is just the two introduction rules `EQI` (Existential Quantifier Introduction) and `CI` (Conjunction Introduction). It is completely separate from the production `PLN/`. (`experimental/pln-inf-ctl/pln-inf-ctl.metta:L608-L725, L1431-L1496`)

**C-4-6 [Production PLN/ does not import chaining]** PLN/'s `lib_pln.metta` (Source 3) imports `library lib_import` and `library PLN lib_pln` only. **Confirmed: production PLN does not consume any `chaining` artifact, including pln-inf-ctl.** chaining's PLN inference-control work has not been integrated downstream. (Source 3 reconciliation §3.1)

## §9 — Cluster-level synthesis

**C-9-1 [Migration table — what URE features survived]**

**chaining is the URE successor for:** the *concept* of pluggable Control + control rules in a separate atomspace + Thompson-sample-named branch selection + Beta-distribution-aware TV machinery (in `common/truthvalue/` but unused by inference-control). Conceptually 5 of URE's ~16 features have surviving traces. The architectural skeleton lives; the Bayesian mathematics largely doesn't.

**chaining did NOT preserve from URE:** BIT (And-Or backward inference tree), MixtureModel, complexity penalty + Kolmogorov K(D), `expansion_andbit_weights`, TraceRecorder, multi-threaded chainer, FCStat, AtomSpace-resident UREConfig, meta-rules (rule-producing rules), AttractionLink + intensional inheritance direct introduction, ImplicationScope quantifier instantiation meta-rules, fuzzy-conjunction-introduction, implication-and-lambda-factorization, conditional-direct-evaluation, closed-lambda-introduction, implication-scope-direct-evaluation, ECAN/STI source-selection. **~15 distinct URE features did NOT migrate.**

**chaining adds beyond URE:** modal-logic chaining (PR #12), lambda-abstraction chaining (PR #4), evolutionary-programming reframed as reasoning (PRs #8-#9), Metamath formalism + propositional calculus port (PRs #14, #22), MORK vs PeTTa benchmarks (PR #25), Problog-inspired probabilistic backward chainer (PR #19), `Continue`-typed control-rule meta-layer with separate atomspace (PR #16, the closest thing to a NEW idea relative to URE — URE had control rules, but the chaining demo shows them as *typed* Continue-relations evaluated by a recursive chainer call, which is a Curry-Howard-friendly variant). **6+ wholly new directions.**

**chaining vs production PLN/ (Source 3):** **Disjoint trees.** Production PLN/ is the lib_pln monolith with PQ-Deriver and the multiplicative-confidence pln0.9 pivot. chaining has its own DTL synthesizer, its own `experimental/pln/` four-encoding pattern, its own `pln-inf-ctl/` mini-PLN dialect. Neither imports the other. **chaining is the research-experimental playground; PLN/ is the production-library line.** Two parallel inheritances of Source 5's seed.

**C-9-2 [Cluster headline question — resolution]** *"Did URE's Bayesian inference-control architecture (Thompson sampling, MixtureModel, control-rule meta-layer, BIT) survive the migration to `chaining`, or was it lost?"*

**Answer:** **Mostly lost; conceptually preserved at one-experiment scale.** The Thompson Sampling function name + the control-rules-in-separate-atomspace pattern are the two surviving concepts. Everything else (the *interesting* mathematical machinery — Beta posterior integration, MixtureModel, complexity penalty, BIT, TraceRecorder, meta-rules) is absent. **The MeTTa-era PLN cluster's "no inference control" gap is mostly intact.** A pedagogical inference-control track exists; a production-grade Bayesian-MAB-plus-meta-rules architecture does not.

This makes the Source-6 reconciliation §6 worry concrete: **URE's Bayesian stack effectively died with URE.** chaining's `pln-inf-ctl` is a tiny new experimental seed of a few of the same ideas, not a port.

**C-9-3 [Wiki implication]** When wiki cards describe PLN's inference-control story, the truthful narrative is:
1. URE (2017-2020) had the Bayesian + BIT + MixtureModel + meta-rule stack (Source 6).
2. URE bit-rotted starting ~2020 with no replacement (Source 6 §5).
3. The MeTTa-era PLN production line (PLN3, PLN/) has no inference control beyond confidence-priority queue (Sources 2-3).
4. **chaining (2025) reattempted a small subset (Thompson-sample boundary cases + control-rule atomspace) at single-experiment scale, then pivoted to Metamath benchmarks.**
5. **The full URE Bayesian stack did NOT migrate to MeTTa.**

**C-9-4 [URE design doc still broken-linked]** Source 6 noted the `examples/pln/inference-control-learning/README.md` design doc (cited from URE's `ActionSelection.h:75`, `ThompsonSampling.h:69`, `MixtureModel.h:84`) as out-of-tree and broken-linked. **chaining inherited the broken link** (`inf-ctl-month-xp.metta:L4` references the same URL pattern, also broken). The design document for this whole architectural lineage is *still missing* and now both URE and chaining point at it without resolving it.

**C-9-5 [common/formula PLN inheritance]** `experimental/common/formula/DeductionFormula.metta` STV deduction-formula uses `min` confidence aggregation — i.e., **chaining preserves the pre-pln0.9 (Source 5 / Source 2 PLN3) `min`-based form**, NOT the pln0.9 (Source 3 `lib_pln.metta`) multiplicative pivot. So if the formulas in `experimental/pln/dependent-types/DeductionDTL.metta` are byte-identical to those at any one of {Source 5, Source 2 PLN3, Source 3 PLN/}: my read suggests **byte-identical to Source 5 / PLN3, not pln0.9**. (Verbatim `(min $Pc (min $Qc (min $Rc (min $PQc $QRc))))` at `experimental/common/formula/DeductionFormula.metta:L52`.)

**C-9-6 [Cluster-narrative refinement]** chaining vs PLN/ are **co-equal descendants of Source 5 hyperon-pln**, not a sequential lineage. Source 4 (pln-experimental2) is also a Source-5 descendant. The "cluster" is now four parallel Source-5 descendants: Source 2 PLN3 (frozen) → Source 3 PLN/ (production), Source 4 pln-experimental2 (research-1), Source 7 chaining (research-2). URE (Source 6) is a separate ancestor whose "successor" claim by chaining is conceptual rather than substantive.
