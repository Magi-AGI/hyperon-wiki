## 3. Claude's "dead infrastructure" finding (X1, important)

[X1 Claude — VERIFIED] Claude uniquely caught: `experimental/common/truthvalue/TruthValue.metta` contains a **richer Beta-distribution / posterior implementation than anywhere else in the cluster** — Jeffreys prior, posterior α/β computation, mean = α/(α+β). But:

- It is **NOT imported by any inference-control file**.
- It is **NOT used by `pln-inf-ctl/pln-inf-ctl.metta`'s `thompson-sample`** (which explicitly avoids it via the c∈{0,1} simplification).
- It is **NOT used in the production PLN line** (Sources 2-3 had no Beta machinery).

This is **dead infrastructure**: someone ported the math, nobody wired it in. Two interpretations:
1. Reserved for future inference-control work that never happened.
2. Pedagogical leftover from the URE-era thinking.

Either way, **the wiki should not claim PLN-cluster has Beta-posterior reasoning** — the math exists but is uncalled.

**Wiki implication**: when documenting `chaining`, distinguish between *defined* and *integrated* code. The TV utilities are defined-but-not-wired; the inference-control concepts are wired-but-toy-scale.

## 6. Wiki implications (cluster-level)

### 6.1 The "no inference control" gap is REAL — flag it

After Source 7, we can confidently state in wiki text:

- **The 2008 PLN book describes BIT + multi-armed bandit control.**
- **URE (OpenCog Classic, C++) implemented this and went beyond it** with Bayesian Thompson Sampling, MixtureModel, control rules, Kolmogorov complexity heuristic, multi-threaded chainer, FCStat, AtomSpace-resident config.
- **`chaining` (the URE successor) preserves only sketches** — `thompson-sample` as a degenerate stub, control rules as 3 hand-written Continue rules in a toy theory. Beta machinery defined but unused.
- **Production PLN3/PLN/ has none of it** — confidence-priority queue only.

The MeTTa-era PLN cluster's inference control is **less sophisticated than the 2008 book described, less than URE implemented, and less than chaining sketches**. This is a legitimate "loss of capability" finding for the wiki.

### 6.2 Cluster architectural narrative for the Published card

Suggested wiki framing for `Hyperon AI Algorithms+PLN+PLN Full` (Draft):

> *"PLN's inference-control mechanism in the current Hyperon-MeTTa lineage is a confidence-priority forward-chaining queue (per `lib_pln.metta`'s `Deriver`). This is a substantial simplification from the 2008 book's described BIT + multi-armed bandit framework, and from the OpenCog Classic URE C++ implementation that extended the book with Bayesian Thompson Sampling, MixtureModel, and AtomSpace-resident control rules. The successor `chaining` repository preserves these concepts at experimental / one-test scale (e.g., `experimental/pln-inf-ctl/`, `experimental/inference-control/`) but has not productionized them into the core `lib_pln.metta` library. Re-establishing inference control at production scale is open research work."*
