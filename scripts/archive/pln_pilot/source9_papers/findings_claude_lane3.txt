### Publications+Guiding PLN with Attention Allocation

**Lane:**            3
**Card type:**       Markdown (stub — abstract only on wiki card)
**Length:**          approx 0.6 KB
**Date / version:**  2014, AGI-14 Springer; authors Harrigan,
                     Goertzel, Iklé, Belayneh
**Wiki updated:**    2026-04-08
**Provenance note:** The wiki card is an abstract stub; full
                     paper text is NOT mirrored on the wiki. All
                     claims below are extracted from the
                     summary/relevance prose only. Claim depth
                     is therefore shallow by source constraint.

**Core claims** (numbered):
  C-1: ECAN ("Economic Attention Networks") provides nonlinear
       dynamical attention allocation as a substrate for
       cognitive resource control.
  C-2: ECAN's attention allocation can guide PLN inference, i.e.
       ECAN is positioned as an external controller modulating
       which PLN inference paths get computational resources.
  C-3: The intended effect is to focus inference on "most
       promising" reasoning paths (truncate combinatorial
       explosion via attention-driven pruning/prioritization).
  C-4: This is framed as directly relevant to PLN+ECAN
       integration in Hyperon's PRIMUS architecture
       (forward-projection by the wiki curator, not the 2014
       authors).
  C-5: The attention-allocator dynamics are nonlinear —
       implying STI/LTI diffusion / Hebbian-style updating
       rather than a static priority queue (ECAN canonical
       design from OpenCog Classic).
  C-6: Attention guidance is positioned as a meta-layer above
       PLN's truth-value math, not a modification of the rule
       set itself.

**Intended implementation target:**
  ECAN attention allocator (STI/LTI/AV) coupled to PLN inference
  driver, controlling per-step expansion choice and resource
  budget allocation across alternative inference branches.

**Current implementation status:**  PAPER-ONLY / ASPIRATIONAL
  in the PLN3/PLN/ cluster.

**Code evidence:**
  - Source 3 Deriver.metta uses confidence-priority queue only
    — no STI/LTI/AV, no diffusion, no Hebbian update.
  - Source 6 URE had a "STI" source-selection-mode hook, but
    that is the URE generation; the actual ECAN math lives in
    the separate opencog/attention repo (NOT in this cluster).
  - Closest in-cluster artifact: chaining/experimental/
    inference-control/ — meta-rule scaffold with 3 hand-written
    Continue-typed CONTROL rules in a months-of-year toy
    theory. This is a structurally analogous experiment
    (meta-control over inference choice) but is NOT ECAN: no
    nonlinear dynamics, no attention values, no diffusion.
  - No production code path implements the 2014 paper's
    proposal in PLN3/PLN/.

**Unresolved questions / cross-references:**
  - Is the chaining/experimental/inference-control/ effort
    intended as a successor/replacement for the 2014 ECAN
    proposal (i.e. MeTTa-native meta-control replacing
    AtomSpace-attached ECAN)? The 3 Continue rules in
    months-of-year suggest YES — the design philosophy moved
    from external dynamical controller to in-language
    meta-rules. Confirmation requires reading the experimental
    README / commit history (not done in this lane).
  - Source 6 URE STI hook vs the 2014 paper: the URE hook
    appears to be the same generation/integration the paper
    describes (Harrigan was on URE-era OpenCog). So C-2 is
    historically IMPLEMENTED in URE/OpenCog Classic, but
    SUPERSEDED for Hyperon — the cluster does not carry that
    code forward.
  - Full paper text would need to be fetched separately
    (Springer/AGI-14 proceedings) to verify protocol-level
    integration claims; the wiki stub does not specify them.

### Publications+Deep Learning Perception with PLN

**Lane:**            3
**Card type:**       Markdown (stub — abstract only on wiki card)

**Date / version:**  2013, AGI-13 Springer; authors Goertzel,

                     Full paper text NOT mirrored. Claims below

**Core claims** (numbered):
  C-1: Deep-learning perception modules produce subsymbolic
       perceptual representations (continuous activations /
       feature vectors).
  C-2: A bridge to PLN's symbolic logical reasoning is achieved
       via FREQUENT PATTERN MINING — specifically the FISHGRAM
       algorithm.
  C-3: FISHGRAM mines patterns over the perceptual stream and
       converts them into PLN-ingestible symbolic predicates
       (atoms / TruthValues) for downstream reasoning.
  C-4: The paper positions itself as a neural-symbolic
       integration prototype; pattern mining is the "interface
       layer" between subsymbolic perception and symbolic
       inference.
  C-5: This is framed as central to Hyperon's neural-symbolic
       integration approach (wiki curator forward-projection,
       not 2013 authors).
  C-6: The architecture is implicitly: encoder (DL) -> pattern
       miner (FISHGRAM) -> PLN — i.e. DL is treated as a
       perceptual front-end whose outputs are mined, not as a
       differentiable continuation of inference.

**Intended implementation target:**
  Neural-symbolic perception pipeline: deep-learning encoder
  produces feature streams → FISHGRAM mines frequent patterns
  → patterns instantiated as Atoms with TVs in AtomSpace → PLN
  reasons over them.

**Current implementation status:**  PAPER-ONLY / ASPIRATIONAL

**Code evidence:**  none — paper-only.
  - Sources 2-3 verified pure-symbolic: no neural model
    imports, no torch/tensorflow/onnx hooks, no perception
    interfaces, no FISHGRAM module visible in the cluster.
  - No frequent-pattern-miner module of any kind in
    PLN3/PLN/ (FISHGRAM was an OpenCog Classic C++ module;
    not ported).
  - No truth-value-from-neural-confidence shim in the
    Deriver / rule code.

**Unresolved questions / cross-references:**

**Core claims** (numbered):

       3-stage architecture (atps/scripts/select_pln_
       mixture_local_nb.py): Stage A multi-bin local experts,
       Stage B normalize-and-pool via ⊕, Stage C tensor-
       compose pooled prior with NB-only likelihood via ⊗.
  C-5: All log-odds/pooling claims hold under EXPLICIT
       DOMAIN HYPOTHESES (finite nonzero channels for log-
       odds theorems; nonzero finite totals for normalization /

  C-8: Empirical: extended MPTP 5k, 4200 train / 800 val. On

  C-10: Methodological commitment: stay in the

CROSS-PAPER COHERENCE
================================================================

The three papers do NOT form a coherent single-arc theme. They
are three independent "PLN as integration substrate" proposals
across three distinct eras and ambition levels:

  - P1 (2014, ECAN+PLN): cognitive-architecture-scale
    integration with a dynamical attention substrate
    (OpenCog Classic era).
  - P2 (2013, DL+PLN via FISHGRAM): perception-layer
    neural-symbolic bridge via pattern mining (pre-LLM era;
    OpenCog Classic era).
  - P3 (2026, kNN/NB+PLN): premise-selection algebra with
    classical ML selectors as algebraic specializations
    (Mettapedia / Hyperon / Lean-certified era).

What they share is a META-STRATEGY: in each paper, PLN is
positioned as the COMMON SUBSTRATE / CARRIER, and the partner
formalism (ECAN attention dynamics; DL+FISHGRAM perception;
kNN/NB classifiers) is reduced or bridged TO the PLN evidence
calculus. P1 and P2 propose this aspirationally; only P3
delivers it formally (Lean) and empirically (MPTP 5k).

Implementation-status one-liner across the 3 papers:
  P1 paper-only / superseded; P2 paper-only / anachronistic;
  P3 fully implemented in adjacent-tree code with Lean proofs
  and 4-round benchmarks — but in atps/+PeTTa/ trees, not
  inside the PLN3/PLN/ Deriver.

FOCUS-QUESTION ANSWERS
================================================================

Q1 (Guiding PLN with Attention Allocation — ECAN integration
   protocol; same as URE STI hook? reflected in
   inference-control/?):
  The wiki card is an abstract stub and does not contain the
  paper's protocol-level detail; the paper itself proposes
  ECAN's nonlinear dynamical attention (STI/LTI/AV) as a
  meta-controller over PLN inference path expansion. The URE
  STI source-selection-mode hook in Source 6 is plausibly the
  same integration generation. chaining/experimental/
  inference-control/ is the closest in-cluster artifact —
  meta-control over inference choice — but it is meta-rule
  based (3 hand-written Continue rules in months-of-year), NOT
  the dynamical ECAN substrate the paper proposes. So:
  conceptually-related successor at toy scale; not a port.

Q2 (Deep Learning Perception with PLN — neural-PLN bridge
   architecture; cluster fit?):
  The paper proposes encoder → FISHGRAM frequent-pattern
  miner → AtomSpace atoms+TVs → PLN. This is a perception-
  front-end pattern, not a differentiable inference path. No
  fit in the cluster: no FISHGRAM port, no neural import, no
  TV-from-neural-confidence shim. Modern Hyperon
  neural-symbolic work (if any) lives outside PLN3/PLN/.

Q3 (PLN kNN NB — what does it combine; classifier-as-TV-source;
   any current code?):
  It expresses Naive Bayes as tensorized likelihood evidence
  and k-NN as local evidence aggregation, both on the same
  E=(n+,n-) PLN evidence carrier. NOT classifier-as-TV-source
  — it is the inverse: classifiers are recovered as
  algebraic specializations of the PLN calculus. Yes, there
  is current code (atps/scripts/select_pln_*.py,
  hyperon/PeTTa/demos/pln_*_selector*.metta,
  hyperon/PeTTa/lib/lib_pln_xi.metta) plus Lean proofs in
  Mettapedia/Logic/ and 4-round MPTP 5k benchmarks. But this
  code lives in adjacent application/PeTTa trees, not inside
  the PLN3/PLN/ Deriver.

Q4 (cross-paper coherence): Three independent applications
   sharing only the meta-strategy "PLN as common substrate."
   See cross-paper section above.

MOST-SURPRISING SINGLE FINDING
================================================================

RawData+Publications+PLN kNN NB is mis-categorized as a
"framing paper" — it is in fact a 2026 engineering paper with
LEAN-PROVEN equivalences (NB and k-NN as evidence-algebra
specializations) and a working flagship selector
(select_pln_mixture_local_nb.py) that ties MaSh NB at 327/800
in 30s top-256 MPTP 5k validation. The paper's central claim
("classical selectors are NB and k-NN specializations on the
same PLN evidence carrier") is not aspirational but
mechanically certified via theorems
toStrength_nbEvidence_eq_nbPosterior and
PLN_hplusPos_eq_knnRelevance. This inverts the standing
correction's framing: kNN/NB are absent from the Deriver, but
they are present (and Lean-bridged to PLN) in the adjacent
PeTTa/atps trees — i.e. there IS a PLN+kNN+NB integration in
the broader Hyperon/Mettapedia ecosystem, just not inside
PLN3/PLN/ proper.
