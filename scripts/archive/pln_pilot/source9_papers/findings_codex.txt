4. Cross-paper synthesis

Most consistent cluster-level ideas
- The FactorGraph/MORK papers consistently assume MORK is a substrate to build on, not a ready-made factor-graph engine.
- The confidence/control lane consistently points toward richer evidence/confidence formalisms than the production `PLN/PLN3` line currently implements.
- Several papers implicitly or explicitly treat “future operationalization” as separate from formal or theoretical articulation.

Major contradictions or mismatches
- FactorGraph/MORK papers assume APIs and runtime primitives that do not exist in Source 8’s MORK code.
- `PLN Chaining On MORK` uses a backward-chaining architecture vocabulary (`RuleAtom`, `GoalAtom`, `ContinuationAtom`) that does not match the FactorGraph family’s `VariableAtom`/`FactorAtom` design.
- The confidence framework and pln0.9 move in a related direction, but the implemented formulas in `PLN/lib_pln.metta` are not derivations of the framework document’s algorithm.
- The application lane’s historical vision (attention-guided reasoning, FISHGRAM-to-PLN) does not map to the current production runtime.

Sibling groupings
- FactorGraph / MORK base + v1 + v2 + v3.1 + v3.2 = one iterative design stream.
- `PLN Chaining On MORK` is adjacent but architecturally distinct.
- `Wm Pln Book V3` + `Pln Review` form a formal-refoundation/audit pair.
- `PLN AND Confidence Framework.md` + `PLN NARS TVs` form a confidence/evidence comparison pair.
- `Guiding PLN with Attention Allocation` + `PLN ECAN 2015v1` form a historical attention-control pair.

2. `PLN Chaining On MORK` should not be merged conceptually into the FactorGraph stream; it is a separate backward-chaining architecture proposal.

4. `Wm Pln Book V3` should be described as a world-model formal refoundation, not as current production PLN runtime.
5. `Pln Review` should be described as a formal audit/proof-review companion, not as evidence that current code has caught up.
6. Attention/ECAN papers should be labeled historical/hybridized: there was prototype resonance in OpenCog Classic, but current production MeTTa PLN lacks that layer.
