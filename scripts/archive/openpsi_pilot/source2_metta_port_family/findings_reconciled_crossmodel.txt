## Cross-model consensus (both Codex and Gemini agree)

### Architecture (A1-A4)

- **Modernized + expanded MeTTa-native reimplementation, not a thin port.** Canonical 67 (Codex local count: 66) `.metta` files vs Classic's 27 — substantial growth and reorganization.
- **Three-layer reorganization:** `main/` (demand, emotion, rules, modulator, mind-agents, feed-back-loop, types) + `psi-utilities/` (logger, miner, normalization) + `use-cases/` (curious-agent, ping-pong, qwestor).
- **Rule representation preserved structurally:** `ImplicationLink` over `(AndLink context action) → goal`, but reimplemented MeTTa-natively with spaces and unification (`main/rules/rule.metta`, `main/rules/implicator.metta`).
- **Satisfiability cache modernized:** MeTTa space `&satisfiabilityCache` replacing Classic's C++ in-memory cache (Codex C2-14, Gemini section 1).
- **5 action-planner variants:** `action-planner-v1` (greedy/weight-based), `action-planner-v2` (hill climbing), `aStar-action-planner` (A*), `thompson-sampling/`, `ts-ucb/`. Replaces Classic's single `psi-get-satisfiable-rules` default. Both models flag this as the most distinctive divergence from Classic.
- **Type system minimal:** `main/types.metta` defines only Goal/Action/Emotion (Codex C2-9). Classic had no equivalent type module.

### ECAN/STI coupling (B1-B4) — Source 1 V1-1 carry-forward CONFIRMED

**Both models verify ECAN coupling is ABSENT in the MeTTa port.** Repo-wide token audit returned zero functional hits for `cog-av-sti`, `cog-set-sti!`, `cog-stimulate`, `cog-af`, `bind-af`, `attention-bank`, `attentional-focus`, `AttentionValue`, `STI`, `LTI`. Both Classic's optional `rule-sca-weight` STI helper and the optional `(opencog attention-bank)` module load are gone — not just dormant but textually absent.

**The Classic action-selector enabler seam did NOT survive as a configurable selector API.** Codex C2-2/C2-30/C2-31/C2-32: the MeTTa port chooses planner behavior by importing specific modules (e.g., `import action-planner-v1` in `curious-agent/main.metta:8` and `qwestor/main-loop.metta:9`) and calling specific functions, not by installing a callback into a global OpenPsi controller. Gemini's softer phrasing ("the seam survives conceptually") is consistent with this once "conceptually" is read as "different mechanism with similar role" — there is still a way to plug a planner into a use-case, but it is not the Classic `psi-set-action-selector!` registry.

**Cluster-narrative alignment:** Source 2 strengthens the Source 1 V1-1 conclusion. Classic OpenPsi was an ECAN-coupling enabler with an unused STI helper; the MeTTa port is a motivation/action-selection subsystem with no ECAN coupling surface at all. ECAN is not "preserved as an optional path" in the MeTTa port — it is structurally gone.
