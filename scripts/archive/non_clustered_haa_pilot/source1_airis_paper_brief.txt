P1. Paper title: "Autonomous Intelligent Reinforcement Inferred Symbolism."

P2. Authors: Berick Cook¹ and Patrick Hammer² (TWO authors, in this order).

    Author ordering is conventional first-author/co-author (Cook first).

P3. Venue: KTH/DiVA portal record diva2:1890142. The paper appears to be
    a stand-alone arXiv-style/preprint format with section numbering,
    figure captions, and a references section; it is not labeled as a
    conference proceedings paper. Reviewers should NOT assume AGI-24
    or other specific venue without explicit paper-text evidence.

P4. Repository citation in paper. Page 2, footnote 3:
    "The current implementation is available in the corresponding
    open-source repository on Github: https://github.com/berickcook/
    AIRIS_Public"
    This is the AUTHORITATIVE paper-internal repository attribution.
    Resolves §3.A in favor of berickcook/AIRIS_Public (option b).

P5. Abstract claim (verbatim): "This paper introduces AIRIS (Autonomous
    Intelligent Reinforcement Inferred Symbolism) to enable causality-
    based artificial intelligent agents. The system builds sets of causal
    rules from observations of changes in its environment which are
    typically caused by the actions of the agent. These rules are similar
    in format to rules in expert systems, however rather than being
    human-written, they are learned entirely by the agent itself as it
    keeps interacting with the environment."
    Keywords: Procedure Learning · Experiential Learning · Autonomous
    Agent · Causal Reasoning · Artificial General Intelligence.

P6. Architecture (§3, page 4, Fig. 1). Four primary components:
    (1) Rule Base — learned rules.
    (2) State Graph — internal world model; current + future predicted
        states.
    (3) Pre-Action Observation — sequence of processes incl. Planning,
        Prediction, Goal Heuristic.
    (4) Post-Action Observation — sequence of processes incl. Prediction
        Verification, Rule Creation, Rule Updating.

P7. Rule structure (§2, page 2; §4, page 5). A Rule's Conditions consist
    of:
      - the action that was taken
      - the Original Value (the value prior to the action)
      - the Relative Change Positions (positions of other values that
        also changed, relative to Original Value's position)
      - the Relative Change Original Values (those values prior to the
        action)
      - a list of other Rules that were observed to be true for the
        prediction
    The Statement is what the Original Value changed into.
    NOTE for S1.Q1: this is paper-formal. The Goertzel 2025-09 SubRep
    boundary paper describes rules as r = ⟨pre(r), act(r), post(r),
    p_r, c_r, g_r⟩ — a 6-tuple including firing probability p_r and
    motive-shift g_r. The Cook & Hammer 2024 paper does NOT formalize
    p_r or g_r. AIRIS has State Confidence (0:1) as the prediction-
    confidence value (§2 "State Graph Creation," page 3), which roughly
    corresponds to c_r. **The Goertzel 2025-09 framing is an extension/
    reframing, not a faithful description of the Cook & Hammer 2024
    formalism.**

P8. State representation. The paper describes inputs as "sub-symbolic
    values which capture the presence of different object types at the
    existing locations of the environment" (§3 Pre-Action Environment).
    For the Grid World Puzzle Game: "a two dimensional array of
    integers that represent the objects in the game world, and a one
    dimensional array of integers that represent the inventory of items
    collected by the agent" (§5 Grid World Puzzle Game, page 6).
    NOTE for S1.Q5: the Cook & Hammer 2024 paper does NOT formalize
    state as the tripartite x = ⟨facts(x), reals(x), focus(x)⟩ that
    Goertzel 2025-09 §2.1 attributes to AIRIS. Focus Values DO exist
    in Cook & Hammer 2024 — but as "values that changed from previous
    State" (§2 State Graph Creation, page 3), a delta-based saliency
    mechanism, NOT as a labeled tripartite state component. **The
    Goertzel 2025-09 ⟨facts, reals, focus⟩ formalization is a Hyperon-
    side reframing not present in the Cook & Hammer 2024 paper.**

P9. State Graph and goal-directed planning (§2, page 3 + §3, page 4).
    "The State Graph represents the system's internal world model, and
    is used to make plans and achieve goals." Goal Sub-States are
    sub-states matching Rule Conditions whose Statement results in
    achieving the agent's specified Goal. Planning traces a path from
    current State to Goal State through the State Graph.
    NOTE for S1.Q6: the Cook & Hammer 2024 paper describes
    DETERMINISTIC rule application + heuristic goal-state search. There
    is NO mention of "Monte Carlo rollouts" anywhere in the paper. The
    Goertzel 2025-09 SubRep paper §2.1 claim that "AIRIS maintains a
    directed state graph G for Monte Carlo rollouts" is **paper-
    divergent** — state graph confirmed, MC rollouts NOT in Cook &
    Hammer 2024.

P10. Curiosity (§2 State Graph Usage, page 3). VERBATIM:
     "If no Goal Sub-States exist or the Goal Sub-States cannot be
     reached, then the State with the lowest Confidence becomes the
     Goal State... This results in the agent pursuing novel situations
     to observe and generate new Rules. ... This can be considered as
     an effective model of artificial curiosity, which does not only
     capture aspects of information gain as in [7] [Still & Precup
     2012], but leads to deliberate interactions planned by the agent
     using the empirically-causal knowledge it possesses."
     **CONFIRMED**: AIRIS curiosity = lowest-State-Confidence-as-Goal-
     State mechanism. Paper explicitly frames this as an artificial-
     curiosity mechanism. This is implementation-backed (puzzle-game
     experiments demonstrate it; §5).

P11. Self-modification scope (LOAD-BEARING for cluster-mapping decision).
     The paper describes AIRIS modifying its OWN Rule Base + State
     Graph based on observations (online rule induction; §3 Rule
     Creation, Rule Updating, page 4-5). It does NOT describe AIRIS
     modifying its own architecture, rule format, planning algorithm,
     or any other meta-level structure. AIRIS is a self-LEARNING
     system, not a self-MODIFYING-CODE system. The wiki boundary card
     `About Hyperon+Self-Modification and Safety` is about a different
     concept (Hyperon-side meta-MeTTa / MeTTa-source-modifying-MeTTa /
     ASI safety literature).
     RECOMMENDATION for S1.Q7: AIRIS belongs as new HAA card, NOT as
     subcard of Self-Mod-Safety. Cross-link only.

P12. Comparisons cited in the paper (§1 Introduction):
     - Pearl's causal networks [9 = Verma & Pearl 1990] — needs
       pre-specified causal model; AIRIS distinct (no pre-spec).
     - Cyc [4 = Lenat et al. 1990] — AIRIS doesn't need prior logical
       knowledge.
     - Non-Axiomatic Reasoning System (NARS) [10 = Wang 2013] — AIRIS
       doesn't demand logical encodings of inputs.
     - PDDL [1] — same.
     - Information-theoretic curiosity [7 = Still & Precup 2012] — AIRIS
       extends beyond information gain.
     - Reinforcement Learning [5 = Moerland et al. 2023, 8 = Sutton &
       Barto 2018] — comparison baselines.

P13. Empirical results (§5 Experiments & Comparisons, pages 5-9).
     - Grid World Puzzle Game (13 levels): untrained agent completes all
       levels in 4,612 timesteps; trained replay in 921; novel level in
       67 timesteps (best possible 54).
     - Cartpole [2 = Geva & Sitte 1993]: AIRIS solved in 130 episodes
       (first 200 score at 32 episodes); DQN baseline 200+ episodes
       (first 200 at 60+ episodes).
     - MNIST: 91% first run, 97% second run, 99% on second test run.
     - All demonstrate data-efficiency claim.

P14. Limitations (§6, page 9). Verbatim: "AIRIS is not able to learn
     rules for events that occur regardless of the agent's action."
     Other limitations:
     - High-dimensional inputs (e.g., video) → computational cost.
     - Will require attention mechanisms to prioritize task-relevant
       values (NOTE for S1.Q11: this is a forward bridge to ECAN-like
       attention allocation).
     - Extension to 3D continuous spaces requires more research.
     - Robot Operating System + Semantic SLAM mentioned as future
       integration.
