S4 does not add a single new unified HAA implementation family. It separates
into:

1. Concept blending: Goertzel-only local paper stack plus a real but
   working-tree-dirty conceptBlending repo. The information-theoretic and
   category-theoretic branches are implementation-backed; quantale,
   paraconsistent-interference, and HoTT material is paper/formal boundary
   material unless later code is found outside this repo.
2. Semantic parsing: two distinct generations. The classical Link Grammar /
   RelEx / learn / generate stack remains AtomSpace/Scheme/C++ oriented. The
   Hyperon-era semantic_parsing / semantic-parser / nl2pln_demo line is LLM to
   PLN/MeTTa, not a wrapper around Link Grammar or RelEx.
3. SGNN: speech-act classification over SwDA, with no Hyperon/Atomese/PLN
   integration in the local repo. Keep as a boundary/defer item, not part of
   S4 semantic parsing or pattern mining.
