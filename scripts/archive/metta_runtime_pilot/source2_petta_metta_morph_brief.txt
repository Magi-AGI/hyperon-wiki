§0. SOURCE IDENTITY
-------------------

This is the SECOND source of the MeTTa runtime cluster pilot. Both sub-sources
are authored or co-authored by Patrick Hammer (patham9) — the same author behind
the closed HAA cluster-pilot Source 1 (Cook & Hammer 2024 AIRIS, where Hammer
is co-author) and HAA cluster-pilot Source 5 (NACE, MeTTa-NARS).

  S2.A — PeTTa. patham9/PeTTa. Efficient MeTTa-language implementation in
         Prolog (per repo README). Active development through 2026-05.
         Includes optional MORK + FAISS spaces integration.

  S2.B — metta-morph. trueagi-io/metta-morph. Macro-based MeTTa-to-(Chicken-)Scheme
         translator. Goal: implement a fast subset of MeTTa as an elegant
         Scheme library, NOT full language capability. Depends on
         hyperon-experimental for the upstream interpreter.

These two sub-sources are bundled into a single brief because:

  - Both are patham9 work with shared design heritage (cross-pilot pattern
    matching with HAA NACE/MeTTa-NARS/AIRIS-co-author per C1-6).
  - Both are MeTTa-runtime alternatives but via paradigm-distinct lineages
    (Prolog interpreter vs Scheme-macro translator).
  - Combined size manageable in a single brief (~259 source files).

Reviewers should produce findings files with explicit per-sub-source sections
(S2.A / S2.B) for clean reconciliation. Estimated findings size ~250-400 lines
per reviewer.
