S2.A.Q1. PeTTa interpreter strategy
-----------------------------------

Verdict: [PETTA-NONDETERMINISTIC-BRANCHING-INTERPRETER]

PeTTa is a Prolog-backed interpreter/compiler hybrid. Its runtime semantics
are nondeterministic through Prolog backtracking and explicit disjunction.

Evidence:

- `src/filereader.pl:20-24` parses top-level forms and classifies function
  definitions vs expressions vs runnables.
- `src/filereader.pl:31-35` translates runnable expressions via
  `translate_expr([collapse, Term], Goals, Result)` and calls the resulting
  Prolog goals.
- `src/filereader.pl:36-44` translates function definitions into Prolog
  clauses and asserts them.
- `src/metta.pl:105` implements `superpose(L, X) :- member(X, L).`
- `src/translator.pl:110-113` translates `superpose` into disjunctive branches
  using `build_superpose_branches` and `disj_list`.
- `src/translator.pl:409-412` builds one branch per `superpose` element.
- `src/translator.pl:50-62` implements runtime `reduce/2`, calling registered
  Prolog predicates where possible and otherwise leaving data or partial calls.
- `src/spaces.pl:44-61` uses Prolog goals and backtracking for `match`.

This is not first-match-wins. Multiple clauses and `superpose` branches can
produce multiple answers. Examples such as `examples/functionhead3.metta:16-20`
and `examples/smartdispatch.metta` exercise multi-result and function/data
dispatch behavior.

S2.A.Q2. PeTTa specialization and translation layer
---------------------------------------------------

Verdict: [HYBRID-COMPILED-INTERPRETED]

PeTTa compiles function definitions into Prolog clauses, translates runnables
into Prolog goal lists, and retains dynamic `reduce` paths for runtime
dispatch and partial calls.

Evidence:

- `src/translator.pl:17-31` translates `(= Head Body)` forms into Prolog
  clauses and stores function metadata.
- `src/translator.pl:94-323` translates MeTTa expressions into Prolog goals.
- `src/translator.pl:300-323` is the automatic smart-dispatch block: known
  functions become direct calls, literals/data are left as data, list heads may
  be recursively evaluated, and unknown heads fall back to runtime `reduce`.
- `src/translator.pl:328-335` chooses an actual function call, partial, or
  specialized call.
- `src/specializer.pl:4-5` wraps specialization with state cleanup.
- `src/specializer.pl:14-57` builds specialized function names and asserts
  specialized translated clauses.
- `src/specializer.pl:97-105` invalidates specializations when a source
  function changes.

So PeTTa is neither a pure interpreter nor a pure ahead-of-time compiler. It is
a translated/interpreted Prolog runtime with optional specialization.

PeTTa has dynamic type lookup with `%Undefined%` compatibility close to S1.

S2.X.Q4. 0.2.x quirk reproduction
----------------------------------

PeTTa Q-1 bind/new-space: [QUIRK-DIVERGENT-SEMANTICS]
PeTTa's `bind!` at `src/metta.pl:232` only matches `['new-state', B]` and
delegates to `change-state!`. Spaces are dynamic Prolog predicates via
`src/spaces.pl`, and examples freely use `bind! &space (new-space)` even
though no PeTTa `new-space` predicate is registered in the core list. This is
not S1's same no-reset behavior; it is a different state/space model.

PeTTa Q-2 cond: [QUIRK-REPRODUCED]
Repo grep finds no MeTTa-level `cond` implementation. PeTTa uses `if` and
`case` in `src/translator.pl:149-174`.

PeTTa Q-3 catchall/nondeterminism: [QUIRK-REPRODUCED]
PeTTa uses Prolog backtracking and `superpose`; examples use catchalls such as
`$_` in `lib/lib_spaces.metta:17` and many large examples. Multi-reduction is
semantic.

metta-morph Q-1 bind/new-space: [QUIRK-FIXED]
`mettamorph.scm:277-285` defines `new-space`, `new-state`, and `bind!`; `bind!`
sets the `vars` hash table entry directly. This can reset a Scheme-side space
binding, unlike S1's tokenizer-binding caveat.

metta-morph Q-2 cond: [QUIRK-REPRODUCED]
There is a Scheme-internal `cond` in `mettamorph.scm:146` inside a helper, but
no MeTTa `cond` macro/operator is exposed. User MeTTa should use `If`/`Case`.

metta-morph Q-3 catchall/nondeterminism: [QUIRK-REPRODUCED]
`mettamorph.scm` maps nondeterminism to `amb`: `superpose` at `:27-32`,
function definition backtracks across registered functions at `:83-93`, and
`Match` uses `amb1` over a space at `:308-316`.

Overall Source 2 verdict
------------------------

PeTTa: active Prolog-backed MeTTa runtime with nondeterministic branching,
translation to Prolog goals, automatic smart dispatch, specialization, broad
stdlib parity, Prolog-style tabling, extended function-removal semantics,
dynamic `%Undefined%`-compatible typing, flat file/git imports, and

MeTTa subset, preserving nondeterminism through `amb` and hygienic macros,

placeholder rather than a working bundled submodule.
