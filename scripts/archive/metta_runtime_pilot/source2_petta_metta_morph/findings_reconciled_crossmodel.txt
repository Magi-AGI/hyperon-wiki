V2-PROTOCOL-1 (Brief-vocab specificity within set)
  Brief vocabulary §7 single-label-per-question is necessary but not
  sufficient. Reviewers should choose the MOST SPECIFIC tag in the set
  that captures their evidence, not the broadest tag. Gemini consistently
  chose less-specific tags in S2 (Q5/Q7/Q8/B-Q1/B-Q2/X-Q1); Codex chose
  more-specific. For S3 dispatch, brief §7 could include "PICK THE MOST
  SPECIFIC TAG THAT FITS THE EVIDENCE" instruction.

V2-PROTOCOL-2 (Gitlink vs submodule discrimination)
  When a directory is empty but `git ls-files -s <dir>` returns mode
  160000, distinguish:
    (a) Properly-registered submodule: `.gitmodules` exists with entry.
    (b) Loose gitlink placeholder: no `.gitmodules`, object may be
        unresolvable.
  Use [METTA-NARS-PLACEHOLDER] (or analogous) for (b), [METTA-NARS-
  EXTERNAL-SUBMODULE] only for (a).

V2-PROTOCOL-3 (Smart-dispatch / branded-feature-name discrimination)
  When a feature is described by a non-exact-token concept (e.g., "smart
  dispatch" as words, not snake_case), do NOT immediately conclude
  "INVALID FRAMING". Search for the concept in comments, examples, and
  README in addition to the exact token. PeTTa's smart-dispatch concept
  was a brief-author drift mitigated by Codex at extraction.

V2-PROTOCOL-4 (Per-repo verdict granularity)
  When a brief specifies per-repo single-label (e.g., S2.X.Q4 "per-repo"
  framing), reviewers MUST provide one tag per repo, not a single tag
  spanning multiple repos. Gemini's S2 Q-2 "[QUIRK-FIXED] in both"
  framing missed the granularity. Codex's per-repo split is the
