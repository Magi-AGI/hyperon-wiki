EXECUTIVE SUMMARY (R1.X consensus statements)
================================================================================

R1.1 — PAPER AUTHORSHIP IS FOUR (not three). Ben Goertzel, Ted Sanders,
       Jade O'Neill, Gino Yu. PDF p.1, both reviewers verified.
       Wiki card 3081 currently lists three authors and must be corrected
       at cluster close.

R1.2 — THE TERM "FISHGRAM" DOES NOT APPEAR IN THE 2013 PAPER. "Fishgram"
       (sentence-case) is the OpenCog Python implementation name used in
       fishgram.py at L57-58 (Pattern class docstring) and L926
       (top-level callsite `fish.outputConceptNodes(layers)`). The all-
       caps "FISHGRAM" is wiki/discussion-board convention. Card 3081's
       summary "Bridges deep learning perception with PLN reasoning
       through FISHGRAM pattern mining" misattributes terminology and
       must be reframed at cluster close to clarify the implementation-
       name vs paper-title distinction.

R1.3 — THE PAPER CITES YUN CHI'S EXTERNAL SOFTWARE; THE OPENCOG CODE IS
       AN IN-HOUSE RE-IMPLEMENTATION. Paper §2 / PDF p.3 says "apply
       Yun Chi's Frequent Subtree Mining software [13]" with footnote 6
       linking nec-labs.com. fishgram.py at b31c7e3 does NOT call or
       vendor Yun Chi's software; it is an in-house AtomSpace forest/
       conjunction miner using local tree.py + adaptors.py machinery.
       Both reviewers concur. The paper-vs-code gap is a separate finding
       distinct from the paper's own claim.

R1.4 — TWO-VERDICT VERDICT (mandated by S1.Q12 split):
       (a) Core toy pipeline (DeSTIN -> Subtree Mining -> PLN axiom
           emission, demonstrated on 7 letter images at paper §3):
           [CODE-REAL-OBSOLETE]. Real OpenCog Python implementation
           existed; deprecated to attic 2019-07-17 (00a766d955a7e46aea
           3b87bc26ec7652a60a4899, Linas Vepstas), removed 2020-06-25
           (147e2206e1463dc49bd64a459e0018ed46753a1f, Linas Vepstas,
           "Remove archaic, unused python code.").
       (b) Full architecture (six-component pipeline including Frequent
           Subtree Recognizer, Perception Collector, Experience DB,
           Semantic Feedback, plus 2013-projected humanoid robot
           integration): [PAPER-ONLY]. Paper §2 / PDF p.3 says
           explicitly "not yet fully implemented"; §2.1 / PDF p.4 says
           Semantic Feedback "is not yet implemented." S1 evidence
           does not show later completion.

R1.5 — DESTIN REPO IDENTITY IS RESOLVED. The paper's cited
       tpsjr7/destin_ted_temp (PDF p.2 footnote 3) and the canonical
       opencog/destin are the SAME LINEAGE. Migration commit
       bbbdcf3 (2013-08-05, "Merge branch 'master' of https://github.com/
       tpsjr7/destin_ted_temp") brought tpsjr7's content into
       opencog/destin. Ted Sanders' commits continued in opencog/destin
       through 2014-04 (e41130c, last). tpsjr7 README still says
       "This repo has been migrated to opencog/destin" (orchestrator
       WebFetch 2026-04-30). NO independent tpsjr7 lineage survived.

R1.6 — FISHGRAM HAS NO HYPERON-ERA SUCCESSOR BY NAME. S3 survey
       confirmed all hyperon-miner-family repos (trueagi-io,
       iCog-Labs-Dev, zariuq) plus rejuve-bio/neural-subgraph-matcher-
       miner have fishgram-pickaxe = 0. opencog/miner + ngeiswei/miner
       have fishgram-pickaxe = 19 in commit messages but ZERO current
       files. The 2013 Python lineage was a clean break, not a
       continuous evolution.

R1.7 — PLN NO-GO THEOREM APPLIES AT NON-TOY SCALE. The 2013 paper's
       toy example dodges the No-Go theorem by feeding only a small
       set of local Inheritance axioms (paper §3.2 / PDF p.7 explicitly
       acknowledges later evidence would change Similarity strength).
       At full scale, semantic feedback and object recognition would
       require joint context/evidence, exactly the case PLN cluster
       pilot's PLNJointEvidenceNoGo.lean addresses.

R1.8 — ECAN-COUPLING VISION IS LEGACY-DECOUPLED. The paper's Semantic
       Feedback proposal (paper §2.1) matches the historical attention-
       guided-vision pattern from the 2014 Harrigan paper and the
       OpenPsi cluster pilot's V4-1 9-event lifecycle. All three
       executable STI couplings (URE forward-chainer, OpenPsi default
       selector, Loving AI Ghost runtime) were deliberately decoupled
       by 2019. No evidence in S1 that the paper's perception-to-
       attention vision survived as executable code.

R1.9 — ADJACENT PAPERS DO NOT BLUR THE FISHGRAM PIPELINE.
       - Card 3108 (2011 CSDLN, Goertzel sole author): conceptual
         predecessor, already disambiguated in card content. Reviewers
         concur.
       - Card 3120 (2010 OpenCog NS Hybrid Neural-Symbolic, Goertzel
         sole author): broader 2010 architectural proposal (HANN
         attractors, MemberLink grounding, NN-activation <-> STI
         conversion). NOT specifically a frequent-subtree-mining ->
         PLN container. See V1-2 dissent below; Codex's framing
         supersedes Gemini's broader claim.

R1.10 — STAGE-2 AUDIT SEED: fishgram.py at b31c7e3 has a likely
        broken import. Line 24: `from logic import PLNviz`. The
        b31c7e3 tree contains no `logic.py` under the audited fishgram
        surface (only `tests/python/old/test_logic.py`). Source 2 must
        confirm runtime viability — if the import was broken at
        b31c7e3, the "Fishgram could be run on this snapshot" assumption
        needs adjustment.

BOTTOM LINE
================================================================================

The 2013 paper's neural-symbolic integration vision was real — paper
was published, OpenCog Python implementation existed, toy pipeline
demonstrated. But it was a vision that never reached its full-scale
execution: the six-component architecture stayed at toy-example state,
Semantic Feedback never executed, and the entire Python lineage was
deprecated to attic in 2019 and removed in 2020.

Hyperon-era pattern-mining work (hyperon-miner family +
neural-subgraph-matcher-miner) is a clean-break paradigm change, NOT
a continuation. The "FISHGRAM" wiki shorthand has caused real
attribution drift on card 3081; cluster-close will clarify
implementation-name vs paper-title.

S2 will audit the actual Classic Fishgram code at b31c7e3 (pre-removal)
to determine whether the toy pipeline was RUNNABLE-OBSOLETE or
BROKEN-OBSOLETE (per CF1.2). S3-S5 build on the clean-break framing
established here and the cross-cluster No-Go / ECAN-decoupling
forwards.

    findings, add curated excerpts).
  - CLAUDE.md § Source-text gaps: remove the FISHGRAM-paper gap entry
