§B. KEY FINDINGS FROM SURVEY (worth flagging in S3 brief)
---------------------------------------------------------

B1. THREE active divergent Hyperon-era miner-family lineages (Codex's
    "split to 5 if survey finds two real active families" trigger):
    (i)  hyperon-miner trio (trueagi-io / iCog-Labs-Dev / zariuq) — MeTTa
         port of pattern-mining. Bidirectional fork-divergence rule applies
         (per memory feedback_bidirectional_fork_divergence.md).
    (ii) opencog/miner + ngeiswei/miner — Classic Scheme/C++ PatternMiner
         still active in 2025. Same naming-collision pattern that triggered
         the "miner-singnet"/"miner-leungmanhin" suffix convention.
    (iii) rejuve-bio/neural-subgraph-matcher-miner — DIFFERENT paradigm
         (neural subgraph matching, not frequent-pattern mining).
    Recommendation for orchestrator: split S3 into S3a (Classical+Hyperon
    symbolic miner family) + S3b (neural-subgraph-matcher-miner) becomes
    a 5-source cluster, OR keep S3 unified with internal Q3a/Q3b split.
    DECIDE WITH USER + REVIEWERS.

B2. FISHGRAM is NOT a Hyperon-era inheritance. fishgram-pickaxe = 0 in
    all hyperon-miner-family repos and in neural-subgraph-matcher-miner.
    The 19 fishgram pickaxe hits in opencog/miner + ngeiswei/miner are
    references in commit messages from when the miner repo was extracted
    from the pre-split monorepo, NOT actual fishgram code in those repos.

B3. opencog/miner + ngeiswei/miner are STILL ACTIVE in 2025 — surprising
    given that the 2019-09-06 AttentionBank removal (commit 318c0b4cb in
    opencog-singnet) supposedly cleared out the Classic ecosystem. The
    miner is operating on AtomSpace independent of AttentionBank/ECAN.
    This is consistent with the ECAN cluster pilot finding that the
    AttentionBank consumers were narrow (Ghost, NLP fuzzy, Python web API,
    OpenPsi) and the AtomSpace itself remained productive.

B4. opencog/vision was active in 2025-06. Last commit "Add working python
    example" — needs investigation in S3.

B5. iCog-Labs-Dev/hyperon-miner has 127+ merged PRs (vs 10 at
    trueagi-io/hyperon-miner). Active development moved to iCog-Labs-Dev.
    The trueagi-io repo may be the upstream-canonical-but-stale pattern,
    similar to opencog/openpsi vs iCog-Labs-Dev/hyperon-openpsi observed
    in OpenPsi cluster pilot.

B6. The DeSTIN code referenced by the 2013 paper at footnote 3 is
    `tpsjr7/destin_ted_temp`, NOT `opencog/destin`. This is "Ted Sanders'
    temp branch" — likely a pre-merge fork. Not in the local reference
    tree. May still exist on GitHub at github.com/tpsjr7/destin_ted_temp.
    S2 should attempt to retrieve or document it as missing.
