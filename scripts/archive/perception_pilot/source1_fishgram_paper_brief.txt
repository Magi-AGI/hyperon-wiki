§3. ORCHESTRATOR-PRE-POPULATED PAPER FINDINGS (verify, do not relitigate)
-------------------------------------------------------------------------
The orchestrator has read the PDF in full. The following paper-internal
facts are documented for reviewers to use as reference; reviewers may
challenge any if they disagree, but do not need to re-extract.

P1. Paper title: "Integrating Deep Learning Based Perception with
    Probabilistic Logic via Frequent Pattern Mining."

P2. Authors: Ben Goertzel (Novamente LLC + HK PolyU), Ted Sanders, Jade
    O'Neill (HK PolyU), Gino Yu (HK PolyU). FOUR authors. Some prior wiki
    references list three; Yu is the 4th.

P3. The term "FISHGRAM" does NOT appear in the paper text. The algorithm
    cited is "Yun Chi's Frequent Subtree Mining software" (paper ref [13]:
    Chi, Xia, Yang, Muntz, IEEE TKDE 2005). The cited download URL was
    http://www.nec-labs.com/~ychi/publication/software.html (footnote 6).

P4. The OpenCog Python "Fishgram" implementation (file: opencog/python/
    attic/learning/fishgram/fishgram.py at b31c7e3) uses sentence-case
    "Fishgram" (Pattern class docstring: "Store a basic pattern and other
    associated data for Fishgram"). The all-caps "FISHGRAM" appears to be
    a wiki-/discussion-board convention not used in either source artifact.
    Authors on fishgram.py per `git log` at opencog-singnet:
      - Jared Wigmore (2012-04 — earliest commits)
      - Keyvan Mir Mohammad Sadeghi (2012-12 → 2013-01)
      - Jade O'Neill (2013-01 → 2013-08; same Jade as paper author #3)

P5. Paper architecture (§2 of paper) names six components between DeSTIN
    and OpenCog:
      1. DeSTIN State DB
      2. Frequent Subtree Miner
      3. Frequent Subtree Recognizer
      4. Perception Collector (normalizes spatial coords, filters
         unchanged percepts, translates to Scheme/Atomese)
      5. Experience DB
      6. Semantic Feedback (NOT IMPLEMENTED at time of writing)

P6. 2013 status, paper §2 verbatim: "This architecture is not yet fully
    implemented, but is a straightforward extension of the current OpenCog
    architecture for processing data from game worlds, and is scheduled
    for implementation later in 2013 in the course of a funded project
    involving the use of DeSTIN and OpenCog for humanoid robot control."

P7. Single experiment in paper §3: 7 letter-form images fed to DeSTIN; 8
    frequent subtrees mined; 17 contained_in evaluations passed to PLN;
    PLN-derived conclusions {Similarity Image1 Image2 <1, .375>;
    Similarity Image3 Image5 <.5, .444>}. Personality parameter k=5.
    Paper acknowledges these conclusions are not useful — toy-data caveat.

P8. PLN translation chain (paper §3.2):
      Evaluation contains(TreeN ImageM)
      ==> Member ImageM (SatisfyingSet (Evaluation contains(TreeN *)))
      ==  Member ImageM hasTreeN  (definition)
      ==> Inheritance ImageM hasTreeN  (M2I rule)

P9. DeSTIN code repo cited at footnote 3: github.com/tpsjr7/destin_ted_temp
    (Ted Sanders' "temp" branch). NOT github.com/opencog/destin. The
    canonical opencog/destin (which is in our local reference tree, frozen
    2016-09) may not be the same lineage as the 2013 paper's experimental
    setup.

P10. Paper says (§1.1) DeSTIN was "designed to work together with an
     accompanying action hierarchy" but no such action hierarchy is
     described in the paper. References "compositional spatiotemporal
     deep learning system, or CSDLN" terminology from paper [10].

P11. The "humanoid robot control" project mentioned as the funding context
     for 2013 implementation: relevant to OpenPsi cluster pilot's Hanson
     dialogue/robotics runtime branch (Source 4 V4-1). Cross-cluster
     boundary check.

P12. Paper acknowledges (§2.1) semantic feedback is critical for "human
     object and event recognition" beyond the 500ms feedforward window,
     but the feedback path was unimplemented as of 2013.
