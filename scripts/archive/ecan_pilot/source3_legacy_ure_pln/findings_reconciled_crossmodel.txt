## Headline answers

### Q1 (highest priority): Was `source_selection_mode::STI` ever implemented?

**A (orchestrator-verified, settles V0-1 dissent):** **Once implemented, then removed/unwired.** The STI source-selection mode was actively wired up in URE forward chaining from **2016-03-09 to 2018-10-23** (~2.5 years), reading atom STI values via `s->getSTI()` for tournament selection. It was commented out on **2018-10-23** with the explicit comment "An attentionbank is needed in order to get the STI...", and the constructor argument was removed entirely on **2019-01-08**, leaving only the dead enum value at `ForwardChainer.h:42` in current HEAD.

**This is the cluster-narrative-deciding finding for Source 3.** Codex's claim (verified by orchestrator) substantively corrects Gemini's "never implemented" reading.

### Q2: Broader URE/PLN ECAN coupling at current HEAD?

**A (consensus):** **None active.** Repo-wide grep across both URE and legacy PLN at current HEADs found no active reads of `AttentionBank`, `AttentionValue`, `AttentionalFocus`, or `cog-bind-af`. `URE:attention-allocation` Scheme hook persists but is **explicitly documented as dead** (`opencog/scm/opencog/ure/ure-utils.scm:1012` "URE:attention-allocation isn't set b/c it isn't in use presently."). `pln-set-attention-allocation` is a thin wrapper around URE's dead hook.

### Q3: URE inference-control architecture inventory

**A (consensus):** URE has a **substantial mature inference-control stack** at HEAD `cec5509`, most of which was simplified or omitted in modern MeTTa `chaining/`. Confirmed components (both models agree):
- **`MixtureModel.cc/.h`** — Universal-Operator-Induction-style aggregation; per-rule beta-distribution likelihood × complexity prior `exp(-cpx_penalty * length)`; weighted average TV
- **`ThompsonSampling.cc/.h`** — distribution over candidate TruthValues; per-candidate beta-PDF × product of others' beta-CDFs; supports direct sample-and-max
- **`BetaDistribution.cc/.h`** — alpha/beta from TV strength × confidence; sample via `boost::math::ibeta_inv(alpha, beta, rng.randdouble())`
- **`ActionSelection.cc/.h`** — Handle-to-TV action maps, Thompson-wrapped
- **`backwardchainer/BIT.h/.cc`** — Backward Inference Tree; `BITNode::operator()` estimates probability-of-usefulness; `AndBIT::expand` + `AndBIT::select_leaf`
- **`backwardchainer/ControlPolicy.cc/.h`** — backward-chaining control-rule selection; uses `MixtureModel` + `ActionSelection`; recognizes `URE:BC:expand-and-BIT` and `URE:BC:preproof-of` patterns
- **`forwardchainer/ForwardChainer.cc/.h` + `SourceSet.cc/.h` + `SourceRuleSet.cc/.h`** — current source weighting via TV fitness only (`complexity_factor * strength * confidence`); Thompson sampling over source-rule pairs
- **`UREConfig.cc/.h`** — current parameters: maximum iterations, complexity penalty, jobs, expansion pool size, retry exhausted sources, full rule application, BC max BIT size, BC mixture-model parameters

**Both models converge** that this is **inference-control code, not ECAN code**. The URE pursued a **separate Bayesian / control-rule meta-learning route** that did not realize the 2009 paper's ECAN-coupling vision.

### Q4: Legacy C++ PLN dependency on URE; ECAN coupling

**A (consensus):** Legacy C++ PLN (`pln-singnet`) is a **URE-dependent rulebase/library**, not a standalone attention-controlled engine. `CMakeLists.txt` requires URE (`FIND_PACKAGE(URE 1.0.0 CONFIG REQUIRED)`). Inference control is entirely delegated to URE. Wrappers `pln-fc`/`pln-bc` invoke URE chainers; `pln-set-attention-allocation` wraps URE's dead `URE:attention-allocation` hook. **No active AttentionBank / AttentionValue / AttentionalFocus reads in legacy PLN source.** Configs repeatedly set `URE:attention-allocation` to `0` / false. Substantial PLN rule + meta-rule surfaces under `opencog/pln/rules/` and `opencog/pln/meta-rules/`, but no ECAN runtime bridge.

### Q5: PLN cluster Source 7 carryover sanity check

**A (consensus):** PLN cluster Source 7 finding ("only ~5 of 16 URE features survived in `chaining/`") is **supported** by Source 3's URE C++ ground-truth inventory. URE has full mature implementations of beta-distribution TV uncertainty, Thompson sampling, action selection, mixture-model control-rule aggregation, BIT backward inference trees, control-policy rule selection, trace/control atomspace plumbing, and forward source/rule pair scoring. **Modern MeTTa `chaining/` sketches some concepts but does not carry the mature C++ stack wholesale.**

**Important caveat (Codex C3-43, C3-44 — Gemini did not flag):** URE's active **test status is degraded**. `tests/ure/CMakeLists.txt` only actively includes `BetaDistributionUTest`, `ActionSelectionUTest`, and `UtilsUTest`. **`ForwardChainerUTest` is commented out**; **backward-chainer tests are disabled** with a CMake comment that code bit-rotted and tests no longer work / URE is no longer supported. Do not cite current URE as a healthy maintained inference-control runtime without this caveat.
