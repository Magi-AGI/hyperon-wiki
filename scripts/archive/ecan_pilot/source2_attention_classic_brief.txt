## Why this source, why now

Source 1 (metta-attention) established that the MeTTa port implements **0/4 strict-literal** paper equations and **1/4 functional analog (wages), 2/4 engineered surrogates (rent, Hebbian), 1/4 absent (stochastic sigmoid)**. The reconciled Source 1 verdict depends on a comparison target: "the 2009 paper's equations." But the 2009 paper is the **specification**; the **implementation baseline** that metta-attention attempted to port is the C++ Classic ECAN codebase.

Source 2's job is to ground that comparison empirically. Three open questions from Source 1 reconciliation:

1. **Is metta-attention a direct continuation of Classic ECAN, or a structurally inspired reimplementation?** Specifically: did the iCog-Labs-Dev cohort port from the upstream `opencog/attention` C++ source, or from the 2009 paper, or some hybrid? File-by-file equation comparison will settle this.
2. **Are the MeTTa "engineered surrogates" simplifications, renamed equivalents, partial ports, or regressions?** For example: does the C++ also use a fund-buffer central-bank wage adjustment, or does it implement the literal `WageStimulus * (sum(pi) / n)` formula?
3. **Was Classic ECAN ever coupled to PLN/URE in executable code, or only in papers?** Source 1 confirmed no ECAN-PLN integration in MeTTa; the analogous question for C++ is required to decide whether the missing control layer in Hyperon's PLN is a Hyperon-era loss or was already absent in OpenCog Classic.

The answer to #2 in particular determines whether the cluster wiki anchor needs a sixth tradition (iCog-Labs-Dev ECAN-on-MeTTa as a distinct line) or just a Tradition 2 extension.

## Comparison targets — equation-by-equation crosswalk to Source 1

The primary deliverable of this source is a **side-by-side equation comparison** with Source 1's MeTTa modules. Use Source 1 reconciliation's reconciled equation table as the comparison frame.

### 1. Rent

**Paper equation (per brief):** `rent = Rent * max(0, log2(si / recentMaxSTI))^2`

**MeTTa surrogate (Source 1):** target-funds/buffer adjustment `f(FUNDS_STI - TARGET_STI, STI_FUNDS_BUFFER) × elapsed-time` at `RentCollectionBaseAgent.metta:7-28, 36-57`. **No `log2`, no `recentMaxSTI`.**

**C++ to verify:**
- `opencog/attention/RentCollectionBaseAgent.cc/.h` — base class
- `opencog/attention/AFRentCollectionAgent.cc/.h` — AF rent
- `opencog/attention/WARentCollectionAgent.cc/.h` — WA rent

**Specific questions:**
- Does `RentCollectionBaseAgent::calculateSTIRent` (or analogous) use the literal `log2(si / recentMaxSTI)^2` formula?
- If yes: confirm metta-attention rent is a regression (lost the literal formula). Cite C++ file:line.
- If no (e.g., C++ also uses fund-buffer adjustment): confirm metta-attention is a direct port of an already-divergent C++ implementation. The 2009 paper would then be the divergent ancestor, not the C++.
- Does C++ track `recentMaxSTI` as a live runtime variable? If yes: where is it updated? (Source 1's `recentVal.metta` updates are commented out.)
- Does WA rent in C++ have the compounding-state bug from `WARentCollectionAgent.metta:30-31`, or was that introduced by the MeTTa port?

### 2. Wages

**Paper equation (per brief):** `wages = WageStimulus * (sum(pi) / n)`

**MeTTa surrogate (Source 1):** `STI_ATOM_WAGE * (1 + clamp((FUNDS_STI - TARGET_STI) / STI_FUNDS_BUFFER, -1, 1)) * stimulus` at `attention-bank.metta:11-37, 49-63, 76-90`. Co-located inside the bank, not a peer agent.

**C++ to verify:**
- `opencog/attentionbank/bank/AttentionBank.cc/.h` — central bank
- Look for `calculateSTIWage`, `calculateLTIWage`, `stimulate` analogues

**Specific questions:**
- Does C++ implement `WageStimulus * (sum(pi) / n)` literally, or does it use the same fund-buffer adjustment as MeTTa?
- Is wages paid by a `WageAgent` (peer scheduled agent) or by a `stimulate()` call on the bank? (Source 1 confirmed metta-attention has no `WageAgent`; if C++ also lacks one, it's a faithful port.)
- Where does `stimulus` come from in C++ — what code path produces stimulation? (In MeTTa: `experiment.metta:73-77` stimulates each input atom per batch.)

### 3. Hebbian update

**Paper equation (per brief):** `c_ij' = ConjDecay * (si * sj / recentMaxSTI^2) + (1 - (si * sj / recentMaxSTI^2)) * c_ij`

**MeTTa surrogate (Source 1):** normalized-STI conjunction with PLN-book revision merge: `conj = norm_i * norm_j + (norm_j - norm_i) * abs(norm_j - norm_i)`, exponentially blended via `TC_DECAY_RATE`, merged via `count_new = (c1+c2) - 0.2*min(c1,c2)`. Located at `HebbianUpdatingAgent.metta:53-60, 77-92, 92, 106-130`. **No `recentMaxSTI^2` or `ConjDecay`.**

**C++ to verify:**
- `opencog/attention/HebbianUpdatingAgent.cc/.h`
- `opencog/attention/HebbianCreationAgent.cc/.h`

**Specific questions:**
- Does C++ implement the literal `ConjDecay * (si*sj/recentMaxSTI^2) + (1 - ...) * c_ij` formula?
- Does C++ borrow PLN-book revision count formula (`(c1+c2) - 0.2*min(c1,c2)`) inline, or is that a metta-attention-specific addition?
- Hebbian creation in C++: same `localToFarLinks 1000.0` hardcoded shadowing pattern, or different?
- Does C++ Hebbian update reference any PLN runtime API (TruthValue revision, `URELink`, etc.) — i.e., is the PLN-book reference code-level integration or just numerical formula borrowing?

### 4. Stochastic diffusion (the most-likely-divergent equation)

**Paper equation (per brief):** `P(diffuse) = 0.5 * (1 + tanh(shape * (si - FocusBoundary)))`

**MeTTa surrogate (Source 1):** ABSENT. The active "stochastic-importance-diffusion" module computes deterministic elapsed-time decay `STI × (1 - decayRate)^averageElapsedTime` at `stochastic-importance-diffusion.metta:22-46, 71-85, 109-118`. The hard AF/WA categorical split substitutes for the smooth focus-boundary probability.

**C++ to verify:**
- `opencog/attentionbank/bank/StochasticImportanceDiffusion.cc/.h` — name suggests literal sigmoid
- `opencog/attention/ImportanceDiffusionBase.cc/.h`
- `opencog/attention/AFImportanceDiffusionAgent.cc/.h`
- `opencog/attention/WAImportanceDiffusionAgent.cc/.h`

**Specific questions (highest priority):**
- Does `StochasticImportanceDiffusion.cc` implement the literal `0.5 * (1 + tanh(shape * (si - FocusBoundary)))` sigmoid?
- If yes: this is the **most important regression** from MeTTa. It means metta-attention dropped the actual stochastic mechanism while keeping the module name. Cite C++ file:line for the `tanh` call.
- If no (e.g., C++ also uses categorical AF/WA split + deterministic decay): the sigmoid was already lost upstream, and the 2009 paper is the divergent ancestor.
- Does the C++ AF/WA split use the smooth probability for boundary crossing, or is it a hard cutoff like MeTTa?

## ECAN-PLN integration audit (highest-leverage deliverable for cluster narrative)

This is **the question that resolves the cluster's "missing control layer" carryover.** Specifically: was Classic ECAN ever coupled to PLN/URE in executable C++ code, or only in OpenCog architecture papers?

**Repo-wide grep targets** (run `rg` directly on the repo, not just file-by-file):

- `\bpln\b` (case-insensitive, word-boundary) — generic PLN references in C++/headers/scm/comments
- `PLN\.h` `pln/` `PLN/` — explicit C++ header includes from a PLN module
- `\bure\b` (case-insensitive) — URE references
- `BackwardChainer` `ForwardChainer` `\bChainer\b` — URE inference engine
- `TruthValue` — does ECAN read/write PLN-style TruthValues? (Yes is expected; this is a sanity check.)
- `mixtureModel` `MixtureModel` — Geisweiller's URE Mixture Model (PLN cluster Source 7 finding: only ~5/16 features survived in MeTTa chaining)
- `Bayesian` `Thompson` — URE Thompson sampling / Bayesian control (PLN cluster Source 7 finding)
- `attention.scm` `default-param-values.scm` — does the Scheme layer load any PLN modules?
- `examples/hopfield/` — does the hopfield example couple ECAN to inference?

**Outcome interpretation:**
- **Zero PLN/URE references in active C++:** confirms ECAN was always architecturally adjacent to PLN, never executable-coupled. The Hyperon "missing control layer" was already absent in OpenCog Classic. Cluster narrative gets simpler.
- **Some references via Scheme glue (e.g., `attention.scm` invokes URE):** ECAN had a Scheme-level integration that was lost in the MeTTa port. This is a discoverable historical control layer.
- **C++-level PLN coupling:** the strongest case. Means ECAN-PLN was once executable and the missing control layer is a Hyperon-era regression.

The cluster wiki anchor's two-paradigm framing depends on this answer.
