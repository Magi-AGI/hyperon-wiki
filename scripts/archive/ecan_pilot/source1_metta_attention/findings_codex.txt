## Headline

metta-attention is a real MeTTa implementation of an ECAN-style attention economy, not merely directory scaffolding. It has working attention-value storage, STI/LTI global funds, wage payment through `stimulate`, attentional-focus maintenance, Hebbian creation/updating, AF/WA rent, AF/WA diffusion, and a "shifting and drifting" experiment.

It does not faithfully implement the four paper equations named in the Source 1 brief. Under a strict literal-formula standard:

1. Rent: NOT literal. The code uses a target-funds/buffer proportional surrogate, not `Rent * max(0, log2(si / recentMaxSTI))^2`.
2. Wages: CLOSEST / FUNCTIONAL, but not a strict literal match to `WageStimulus * (sum(pi) / n)`. The code uses `stimulus * calculateSTIWage()`, where `calculateSTIWage` is a clamped central-bank fund adjustment.
3. Hebbian update: NOT literal. The code uses normalized min/max STI plus a PLN-book revision merge, not `ConjDecay * (si * sj / recentMaxSTI^2) + ...`.
4. Stochastic diffusion: ABSENT. The file named `stochastic-importance-diffusion` implements deterministic bin elapsed-time decay; there is no `tanh`, sigmoid, or stochastic focus-boundary probability.

The PLN-cluster "missing control layer" gap remains open. This repo is self-contained ECAN infrastructure and experiments; it is not wired to `PLN/`, `PLN3`, `CeTTa`, or `chaining`.

### Attention value and funds accounting

Path: `attention-bank/attention-value/getter-and-setter.metta`
LOC: 495
Status: IMPLEMENTED attention-value store and funds bookkeeping

Evidence:

- `getter-and-setter.metta:16-31` sets or replaces `(AV sti lti vlti)` and bin state.
- `getter-and-setter.metta:36-45` computes `stiDiff = oldSti - sti`, `ltiDiff = oldLti - lti`, updates funds by adding those diffs, then calls `attentionValueChanged`.

Conservation reading:

However, this is an operational accounting convention, not a proved invariant. There are no tests/assertions that total atom STI plus `FUNDS_STI` is globally conserved after a full agent cycle, and the extrema updater is monotone/one-sided rather than a full recomputation.

`WARentCollectionAgent.metta:30-31` mutates `StartingAtomStiRent` and `StartingAtomLtiRent` every time `collectRent` runs. This looks like a real stateful bug unless the intended meaning is to globally redefine starting rent after every WA rent event.

## Conservation-Law Audit

The repo implements real currency bookkeeping, but not a proved or exhaustively tested conservation law.

Evidence:

- `setAv` returns funds when atom importance decreases and spends funds when atom importance increases (`getter-and-setter.metta:36-45`).
- `attentionValueChanged` writes updated `FUNDS_STI` and `FUNDS_LTI` (`getter-and-setter.metta:69-75`).
- `stimulate` pays wages from the fund-backed `setAv` path (`attention-bank.metta:76-87`).
- Rent deducts atom STI/LTI and therefore returns funds through the same `setAv` path (`AFRentCollectionAgent.metta:82-90`; `WARentCollectionAgent.metta:34-44`).
- Diffusion computes balanced transfers with `tradeSti` / `tradeList` (`ImportanceDiffusionBase.metta:303-320`, `:344-369`) and applies final STI values through `setSti`, which calls `setAv` (`getter-and-setter.metta:228-241`).

Caveats:

- There is no total-mass invariant assertion in the source or tests.
- The runner uses `hyperpose`, several agents use `bach-p`, and parameter updates use `remove-atom` plus `add-atom`; this makes ordering/concurrency behavior non-obvious.
- WA rent mutates global rent-base parameters (`WARentCollectionAgent.metta:30-31`), which can distort the economy across invocations.

## AF vs WA Variant Comparison

Both variants are implemented, but they differ in quality and risk.

AF rent:

- Selects all AF atoms (`AFRentCollectionAgent.metta:20-21`).
- Applies rent by elapsed-time threshold/frequency (`AFRentCollectionAgent.metta:31-56`).
- Does not mutate rent-base global params.

WA rent:

- Selects one random non-AF atom (`WARentCollectionAgent.metta:5-7`; random source in `attentional-focus.metta:45-50`).
- Uses bin elapsed time (`WARentCollectionAgent.metta:24`).
- Mutates `StartingAtomStiRent` and `StartingAtomLtiRent` globals (`WARentCollectionAgent.metta:30-31`), likely a compounding bug.

AF diffusion:

- Uses top-k AF selection and splits source STI by incident/Hebbian paths (`AFImportanceDiffusionAgent.metta:69-97`).

WA diffusion:

- Selects one random non-AF atom (`WAImportanceDiffusionAgent.metta:6-25`).
- Computes decay delta via `diffusedValue` and splits it over incident/Hebbian paths (`WAImportanceDiffusionAgent.metta:59-75`).

Conclusion: AF and WA are both real, but WA rent is suspicious and the stochastic-probability mechanism is absent in both.
