## Headline answer (cluster-pilot question)

**Q: Has the ECAN attention-currency mechanism (rent / wages / Hebbian update / diffusion stochastic from the 2009 ECAN paper) actually been rebuilt in MeTTa, or is `metta-attention/` a structural scaffold that mirrors OpenCog Classic ECAN at the directory level without realizing the equations? And does it close the PLN cluster's "missing control layer" gap?**

**A (three-model consensus):** metta-attention is a **real partial ECAN runtime, not scaffolding** — working STI/LTI currency bookkeeping, attention-value store, attentional-focus maintenance, Hebbian creation/updating, AF/WA rent, AF/WA diffusion, and a runnable "Shifting and Drifting" experiment. It is **not** a verbatim port of the 2009 paper math: under strict literal-formula matching, **0 of 4 brief equations are implemented exactly**; under "functional analog with the right sign," **1 of 4** is materially close (wages, via clamped fund-buffer surrogate). The remaining three are engineered surrogates (rent uses target-funds/buffer; Hebbian uses normalized-STI conjunction with PLN-book revision merge) or absent (the focus-boundary `tanh` sigmoid).

**The PLN-cluster "missing control layer" gap is NOT closed by this source.** All three models confirm zero ECAN-PLN runtime integration in the import graph: no `PLN.Query`, no `lib_pln`, no `chaining`, no `CeTTa` references in active `.metta` source. metta-attention is organizationally distinct (iCog-Labs-Dev / SingularityNET Ethiopia cohort) from the trueagi-io PLN line and the zariuq ai-agents/CeTTa line, which supports the structural-isolation hypothesis from the brief.

## Universal agreement (3-of-3 consensus)

All three models converge on the following findings. These should be treated as the canonical Source 1 result.

1. **metta-attention is a real partial ECAN runtime, not scaffolding.** Working currency-economy with STI/LTI fund accounting, attention-value store, attentional-focus maintenance, AF/WA rent, AF/WA diffusion, Hebbian creation/updating, and a "Shifting and Drifting" experiment.

2. **Zero ECAN-PLN runtime integration.** No `PLN.Query`, `lib_pln`, `\bpln\b` (active code), `\bure\b`, `chaining`, or `CeTTa` references in active `.metta` source. The only PLN reference is `PLN_BOOK_REVISION` as a **local merge-formula tag** inside `HebbianUpdatingAgent.metta` — conceptual borrowing of a PLN-book truth-value revision equation, not a runtime invocation. The PLN-cluster missing control layer **remains open**; this source does not close it.

3. **`stochastic-importance-diffusion` is misnamed.** The active code at `stochastic-importance-diffusion.metta:22-46, 71-85, 109-118` implements **deterministic** elapsed-time decay `STI × (1 - decayRate)^averageElapsedTime`, not the paper's `0.5 * (1 + tanh(shape * (si - FocusBoundary)))` stochastic focus-boundary probability. The "stochastic" name is inherited concept naming from the paper; the runtime mechanism is deterministic.

4. **`WARentCollectionAgent` has a probable compounding-state bug at lines 30-31.** It reads `StartingAtomStiRent` and `StartingAtomLtiRent` from globals, scales by elapsed time, then **writes the scaled values back into the global parameters**. Subsequent `collectRent` invocations therefore compound the elapsed-time scaling. `AFRentCollectionAgent` does not have this issue — it reads at lines 71-75 and computes locally.

5. **Parameter shadowing: `LOCAL_FAR_LINK_RATIO` (10.0 in `AttentionParam.metta:15`) is shadowed by hardcoded `localToFarLinks = 1000.0` in `HebbianCreationAgent.metta:17`.** The active value is 100× larger than the declared parameter, suppressing far-link creation for smaller AF sizes.

6. **`ForgettingAgent` is excluded from the canonical agent loop and from the bulk test runner.** `runner-Def.metta:10` comments out `(forgettingAgent-Run (atomBin))`. `scripts/run-tests.py:78-91` excludes ForgettingAgent tests from the test suite. The implementation itself is non-trivial (`ForgettingAgent.metta:6-25, 32-50, 58-74, 172-183, 210-214`) — this is **dead working code**, not absent code.

7. **Dead infrastructure pattern.** `recentVal.metta:22-67` has both `updateMin` and `updateMax` implementations **commented out**. `getMaxSTI`/`getMinSTI` in `importance-index.metta:201-212` actively read `MAX_STI`/`MIN_STI` from `AttentionParam`, while the commented variants at `:165-189` would have used `recent_max`/`recent_min`. This matters because **the paper formulas the brief asks about depend on `recentMaxSTI`** — the active runtime cannot literally implement those formulas because the `recentMaxSTI` infrastructure is dormant. `DIFFUSION_TOURNAMENT_SIZE` and `RENT_TOURNAMENT_SIZE` are similarly defined parameters with **no active reader** in source.

8. **Both AF and WA variants of rent and diffusion are real, not stubs** — but they implement a hard categorical AF-vs-WA split rather than the paper's smooth focus-boundary probability.

9. **Author cohort is iCog-Labs-Dev / SingularityNET Ethiopia.** Birhane Gulilat (BlightG) is the dominant contributor; the team is organizationally distinct from `trueagi-io` (PLN production line) and `zariuq` (ai-agents/CeTTa line).

10. **Tests not run.** Repo expects sibling `PeTTa` checkout and invokes `sh run.sh` from PeTTa (`scripts/run-tests.py:63-70, 106-113`; README `:21-50`). Per brief, models did not run tests.

## Conservation-law audit (reconciled)

- **STI/LTI bookkeeping is real**, not notional. `setAv` at `getter-and-setter.metta:36-45` decreases `FUNDS_STI` when atom STI rises and returns funds when atom STI falls.
- **No proved or asserted total-mass invariant.** No tests check `sum(atom_STI) + FUNDS_STI = constant` after a full agent cycle.
- **Concurrency complications.** Runner uses `hyperpose` (parallel agents); parameter updates use `remove-atom` + `add-atom` non-atomically; `AttentionParam` updates can race.
- **WA rent compounding bug** (`WARentCollectionAgent.metta:30-31`) breaks any candidate invariant by mutating rent-base globals.

**Reconciled position:** functional currency economy, **not a verified conservation law**. Wiki claims about ECAN as "currency-conservation engine" should be downgraded accordingly.
