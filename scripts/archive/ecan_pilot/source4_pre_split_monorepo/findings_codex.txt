## Executive verdict

C4-01. Source 4 confirms the Source 3 ECAN-PLN/URE narrative, with one refinement:

- No additional active PLN runtime coupling to AttentionBank exists at the `b31c7e3` monorepo snapshot.
- `cog-bind-af` was historically consumed by PLN unit tests, but Nil Geisweiller replaced those calls with `cog-execute!` on 2019-07-04 because "attention focus is irrelevant for these unit tests."
- Therefore, beyond the URE STI source-selection hook from Source 3, I found no durable executable ECAN-PLN inference-control bridge.

C4-02. Source 4 also shows ECAN/AttentionBank was not an orphan in the broader monorepo. At `b31c7e3`, active non-PLN consumers existed:

- OpenPsi action selection used `cog-av-sti` in rule weighting.
- Ghost rule/action selection used `cog-av-sti`, `cog-af`, and `cog-set-sti!`.
- NLP fuzzy matching used `AttentionBank` and `atom_is_in_AF`.
- Python web APIs exposed AttentionValue / AttentionalFocus read-write surfaces.

C4-03. Cluster-level impact: the "only executable ECAN-PLN/URE coupling was the narrow URE STI source-selection hook" remains true if scoped to PLN/URE. It is false if interpreted as "AttentionBank had no executable consumers." The correct wording must distinguish PLN/URE from broader OpenCog consumers.

C4-04. The 2019-09-06 AttentionBank removal deleted ECAN's core modules, tests, Scheme bindings, and Cython wrapper, but left some higher-level consumers elsewhere in the monorepo at or near that transition. This supports the view that AttentionBank removal was broad subsystem removal, not just cleanup of an unused directory.
