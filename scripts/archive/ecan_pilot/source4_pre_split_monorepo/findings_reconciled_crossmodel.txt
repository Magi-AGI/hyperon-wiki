## Reconciled cluster-narrative position (FINAL for extraction phase)

The ECAN cluster pilot extraction phase narrative, after Sources 1-4:

### PLN/URE-scoped narrative (locks in)

> The only verified executable ECAN-PLN/URE coupling beyond unit tests was the **narrow URE forward-chainer STI source-selection hook**, active **2016-03-09 to 2018-10-23**. The hook read atom STI values via `s->getSTI()` for tournament selection of inference sources; it did not run ECAN agents, did not couple PLN reward signals to ECAN currency dynamics, and did not realize the 2009 paper's full ECAN-driven inference control vision. Deliberately unwired by Nil Geisweiller in commit `0b744dbab` (2018-10-23 URE rewrite) with the explicit comment "An attentionbank is needed in order to get the STI..." — the 2018 rewrite authors explicitly knew the AttentionBank dependency was required and chose not to wire the new `SourceSet` path through it. The constructor argument was removed in commit `5a5b7785d` (2019-01-08), leaving only the dead enum at `ForwardChainer.h:42` in current HEAD. URE replaced the old STI hook with internal **TV-fitness × `exp2(-complexity_penalty × complexity)` source weighting in `SourceSet`** (`SourceSet.cc:38-54, 217`); separately, the backward/control-rule `MixtureModel` uses an exponential complexity prior (`exp(-cpx_penalty × length)`, `MixtureModel.h:88`-area). Both are local URE control mechanisms, decoupled from any global AttentionBank.
>
> Source 4 confirms this scope: at the pre-removal monorepo snapshot `b31c7e3` (2019-09-04), `opencog/pln/` had no direct AttentionBank coupling — only a config flag `URE:attention-allocation` set to `0`. PLN-related `cog-bind-af` was unit-test-only and replaced by `cog-execute!` in `0064bfab` (2019-07-04 Nil Geisweiller) with the explicit comment "since attention focus is irrelevant for these unit tests." `pln-set-attention-allocation` is a thin wrapper around URE's already-dead flag, introduced post-monorepo in `86b433fea` (2020-03-30) and never matured.
>
> The "missing inference-control layer" in modern Hyperon PLN is **not a Hyperon-era regression**. It is a 2018 OpenCog Classic deliberate removal during URE modularization, predating the MeTTa transition by 4+ years. The 2009 paper's ECAN-PLN aspirations were realized at the executable layer only as the narrow STI source-selection hook; the richer ECAN-driven inference control loop the paper described was always paper-coupling and architectural aspiration.

### Broader-OpenCog scope (additions)

> ECAN/AttentionValue had real executable use beyond PLN/URE in the 2017-2019 OpenCog era: **OpenPsi** weighted action-selection rules by `strength × confidence × STI` (`action-selector.scm:63`, since at least 2017-11-09); **Ghost** dialogue rule/action selection used `cog-av-sti`, `cog-af`, and `cog-set-sti!` actively in `matcher.scm`, `schemas.scm`, and `stimulation.scm`; **NLP fuzzy matching** included `AttentionBank.h` directly and used `bank->atom_is_in_AF` for AF-only fuzzy search (`Fuzzy.cc:321`, since 2016-12-29 commit `12673ff10`); **Python web API** exposed REST AttentionValue read/write surfaces (`apiatomcollection.py`).
>
> The 2019-09-06 AttentionBank removal commit `318c0b4cb` ("First rough cut at removing the attentionbank") was an ecosystem-disruptive subsystem deletion that removed agent modules, AttentionBank, AttentionSCM, AttentionValue, StochasticImportanceDiffusion, tests, and Cython wrapper. Source 4 found these consumers were **not preserved as functioning ECAN integrations across the removal** — some (Fuzzy.cc) still referenced the deleted headers post-removal in stale, broken form rather than being cleanly migrated or cleanly absent. Ghost, OpenPsi, and the NLP/Python web-API consumers are out of scope for the metta-attention port; no documented MeTTa equivalents were found.
