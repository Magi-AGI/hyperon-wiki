## Executive Verdict

C2-01. Classic `attention/` is the direct implementation baseline needed to interpret `metta-attention/`. The current C++ code does **not** implement the 2009 ECAN paper formulas literally either. Source 1's MeTTa code is therefore mostly a port of the later Classic C++ implementation, not a regression from a fully literal C++ baseline.

## Rent

C2-12. Classic C++ rent is not the 2009 paper's logarithmic rent formula. There are no `log2` hits in the active rent implementation.

C2-13. `RentCollectionBaseAgent::load_params` reads starting STI/LTI rent, target funds, and buffer sizes from ECAN params at `RentCollectionBaseAgent.cc:64-74`.

C2-14. STI rent is a homeostatic funds-gap rule: `diff = targetSTI - funds`, early return `0` if funds are already at/above target, `ndiff = diff / stiFundsBuffer`, clamp to `[-0.99, 1.0]`, return `STIAtomRent + STIAtomRent * ndiff` (`RentCollectionBaseAgent.cc:76-92`).

C2-15. LTI rent uses the same funds-gap pattern without the STI early-return path: `diff = targetLTI - funds`, `ndiff = diff / ltiFundsBuffer`, clamp to `[-1.0, 1.0]`, return `LTIAtomRent + LTIAtomRent * ndiff` (`RentCollectionBaseAgent.cc:95-103`).

C2-16. AF rent selects all atoms in attentional focus via `get_handle_set_in_attentional_focus` (`AFRentCollectionAgent.cc:53-57`), gates by update frequency (`:67-74`), scales rent by elapsed-time multiplier `w` (`:76-84`), clamps rent to current STI/LTI (`:86-87`), then writes updated STI/LTI (`:89-90`).

C2-17. WA rent selects one random atom outside AF (`WARentCollectionAgent.cc:51-57`), computes elapsed time via `StochasticDiffusionAmountCalculator::elapsed_time` (`:65`), then mutates member fields `STIAtomRent = STIAtomRent * last_update_time` and `LTIAtomRent = LTIAtomRent * last_update_time` before calling the base rent calculation (`:68-74`).

C2-18. The Source 1 WARentCollectionAgent compounding-state bug is inherited from Classic C++, not introduced by the MeTTa port. The C++ implementation mutates the rent parameters in place on every WA rent event.

## Wages and Conservation

C2-19. Classic wages are implemented in `AttentionBank`, not as a peer agent.

C2-20. `AttentionBank::stimulate` computes `stiWage = calculateSTIWage() * stimulus` and `ltiWage = calculateLTIWage() * stimulus`, creates a new AV with increased STI/LTI, updates the importance index, and calls `AVChanged` (`AttentionBank.cc:142-157`).

C2-21. `calculateSTIWage` is homeostatic: `diff = funds - targetSTI`, `ndiff = diff / stiFundsBuffer`, clamp to `[-1.0, 1.0]`, return `STIAtomWage + STIAtomWage * ndiff` (`AttentionBank.cc:168-177`). LTI follows the analogous rule (`:179-187`).

C2-22. This is not literal `WageStimulus * sum(pi) / n`. It is a functional analog: stimulus converts to STI/LTI wages, with the conversion rate adjusted by bank funds.

C2-23. AttentionBank enforces local accounting by updating funds when AVs change: it adds old STI/LTI and subtracts new STI/LTI (`AttentionBank.cc:112-127`), then updates importance index and AF (`:128-139`).

C2-24. Starting funds and wage/fund defaults are hardcoded in `AttentionBank` constructor: funds `100000`, buffers `10000`, targets `10000`, wages `10`, max AF size `100` (`AttentionBank.cc:35-45`).

C2-25. Classic documentation explicitly frames STI/LTI as conserved currency: the README says wages pay atoms in use, rent is collected, and AtomSpace funds are managed to remain in a homeostatic range (`opencog/attention/README:106-174`).

## Hebbian Updating and Creation

C2-26. Classic Hebbian updating is not the 2009 paper's literal equation. It uses a target-conjunction surrogate based on normalized STI and then merges a SimpleTruthValue.

C2-27. `HebbianUpdatingAgent::run` chooses one random source atom from attentional focus and calls `updateHebbianLinks(source)` (`HebbianUpdatingAgent.cc:52-72`).

C2-28. `updateHebbianLinks` uses fixed `tcDecayRate = 0.1`, computes `new_tc = targetConjunction(outgoing)`, old mean from the existing TruthValue, and `tc = tcDecayRate * new_tc + (1.0 - tcDecayRate) * old_tc` (`HebbianUpdatingAgent.cc:74-91`).

C2-29. The new TruthValue is `SimpleTruthValue::createTV(tc, 0.1)` and is merged via `h->getTruthValue()->merge(newtv)` (`HebbianUpdatingAgent.cc:92-95`). This is AtomSpace TruthValue machinery, not a PLN runtime dependency.

C2-30. `targetConjunction` normalizes both endpoint STIs with `getNormalisedZeroToOneSTI(..., true, true)`, computes `norm_i * norm_j + (norm_j - norm_i) * abs(norm_j - norm_i)`, then maps it through `(conj + 1.0) / 2.0` (`HebbianUpdatingAgent.cc:99-115`).

C2-31. Classic Hebbian creation reads `MAX_LINKS` and `LOCAL_FAR_LINK_RATIO` from `AttentionParamQuery` (`HebbianCreationAgent.cc:71-77`), with default `LOCAL_FAR_LINK_RATIO = 10` (`default-param-values.scm:61-64`).

C2-32. Source 1's hardcoded `localToFarLinks = 1000.0` is a MeTTa-port regression/shadowing issue. Classic C++ does not have that hardcoded value in `HebbianCreationAgent`; it reads the param.

C2-33. Classic Hebbian creation links AF neighbors bidirectionally where needed (`HebbianCreationAgent.cc:116-134`), creates far links based on `round(count / localToFarLinks)` (`:136-147`), and initializes new links with `SimpleTruthValue(0.5, 0.1)` plus VLTI increment (`:172-177`).

## Importance Diffusion and Stochastic Diffusion

C2-34. Classic C++ does not contain the 2009 paper stochastic sigmoid formula `0.5 * (1 + tanh(shape * (si - FocusBoundary)))`. Repo grep over active code returned no `tanh` or `sigmoid` hits for the diffusion decision formula.

C2-35. The file named `StochasticImportanceDiffusion` implements bin-based elapsed-time estimation and exponential decay, not stochastic diffusion selection by sigmoid.

C2-36. `StochasticDiffusionAmountCalculator` groups atoms into importance bins, records count/update rate/last-update metadata (`StochasticImportanceDiffusion.h:49-60`), and updates bins at `StochasticImportanceDiffusion.cc:49-75`.

C2-37. `elapsed_time` returns `2 / update_rate` for the bin if available, then updates the bin (`StochasticImportanceDiffusion.cc:115-130`).

C2-38. `diffused_value` computes `get_sti(h) * pow((1 - decay_rate), average_elapsed_time)` (`StochasticImportanceDiffusion.cc:140-145`).

C2-39. Source 1's "stochastic diffusion" misnaming is inherited from Classic C++ naming. The implementation is deterministic elapsed-time decay with stochastic source sampling in WA diffusion, not a literal sigmoid transition probability.

C2-40. AF diffusion loads `MAX_SPREAD_PERCENTAGE`, `HEBBIAN_MAX_ALLOCATION_PERCENTAGE`, and `SPREAD_HEBBIAN_ONLY`, then spreads importance (`AFImportanceDiffusionAgent.cc:44-55`).

C2-41. AF diffusion source vector is all AF atoms via `ImportanceDiffusionBase::diffusionSourceVector`, diffused in a loop, then the diffusion stack is processed (`AFImportanceDiffusionAgent.cc:61-72`).

C2-42. AF diffusion amount is `get_sti(h) * maxSpreadPercentage` (`AFImportanceDiffusionAgent.cc:79-82`).

C2-43. WA diffusion selects one random atom outside AF (`WAImportanceDiffusionAgent.cc:83-92`) and computes amount as `get_sti(h) - sdac.diffused_value(h, maxSpreadPercentage)` (`:101-107`).

C2-44. Potential Classic bug: `WAImportanceDiffusionAgent::run` assigns `hebbianMaxAllocationPercentage` from `AttentionParamQuery::dif_tournament_size` (`WAImportanceDiffusionAgent.cc:48-52`). The variable name suggests it should read `HEBBIAN_MAX_ALLOCATION_PERCENTAGE`, as AF diffusion does. Reconciliation should verify whether this is intended or a bug.

C2-45. `ImportanceDiffusionBase::diffuseAtom` builds incident and Hebbian-adjacent probability vectors, combines them, calculates total diffusion amount, and pushes events (`ImportanceDiffusionBase.cc:80-124`).

C2-46. Actual STI transfer is conservative at event level: subtract from source and add to target (`ImportanceDiffusionBase.cc:200-204`).

C2-47. Incident diffusion probability is uniform over incident atoms (`ImportanceDiffusionBase.cc:295-309`).

C2-48. Hebbian-adjacent probability discounts max allocation by link strength/confidence (`ImportanceDiffusionBase.cc:319-352`, `:458-465`).

C2-49. Combined diffusion allocates a configured share to Hebbian-adjacent atoms and leftover to incident atoms; debug assertions check sum near 1.0 (`ImportanceDiffusionBase.cc:369-442`).

## Recent STI / Importance Index

C2-50. Classic C++ has live recent min/max infrastructure, unlike Source 1 where `recentMaxSTI`/`recentMinSTI` handling was found commented out.

C2-51. `ImportanceIndex.h` stores `_maxSTI` and `_minSTI` as `opencog::recent_val` (`ImportanceIndex.h:58-60`).

C2-52. `ImportanceIndex::update` recomputes min/max seen from bins and writes `_minSTI`/`_maxSTI` (`ImportanceIndex.cc:110-135`).

C2-53. `getMaxSTI(true)` and `getMinSTI(true)` return the `recent` smoothed values; false returns raw current values (`ImportanceIndex.cc:139-157`).

C2-54. This matters because Hebbian updating calls `getNormalisedZeroToOneSTI(..., true, true)`, so Classic has smoothed normalization infrastructure available for Hebbian surrogates. It still does not realize the paper rent formula.

## Agent Loop / Runtime Wiring

C2-55. Classic registers and creates all canonical agents in `AttentionModule::init`: AF/WA importance diffusion, AF/WA rent collection, Forgetting, HebbianCreation, HebbianUpdating (`AttentionModule.cc:88-110`).

C2-56. The canonical `do_start_ecan` starts only four agents: AFImportanceDiffusion, WAImportanceDiffusion, AFRentCollection, WARentCollection (`AttentionModule.cc:118-130`).

C2-57. `do_start_ecan` has ForgettingAgent, HebbianCreationAgent, and HebbianUpdatingAgent start calls commented out (`AttentionModule.cc:132-135`).

C2-58. `do_stop_ecan` stops all seven agent pointers (`AttentionModule.cc:141-154`), which confirms the non-started agents are known to the module but excluded from default startup.

C2-59. Source 1's runner differs from Classic default behavior. Source 1 runs Hebbian update/creation in its canonical runner while commenting out Forgetting; Classic default starts neither Hebbian nor Forgetting, although all three are registered and tested.

C2-60. Tests include ForgettingAgent, ImportanceDiffusion, HebbianCreation, HebbianUpdating, AF/WA diffusion, AF/WA rent, and RentCollectionBase (`tests/attention/CMakeLists.txt:7-18`). Classic has stronger test coverage for non-started agents than Source 1's runner behavior would imply.

C2-73. Rent: Source 1's funds-gap rent code and WA compounding bug should be classified as a Classic-port behavior, not a MeTTa-only invention.

C2-74. Wages: Source 1's attention-bank wage implementation should be classified as a Classic-port functional analog. There is no separate WageAgent in either implementation.

The target-conjunction surrogate is Classic-derived; the specific `PLN_BOOK_REVISION` formula name/string and arithmetic detail in MeTTa is a Source 1 artifact, but it does not create runtime PLN integration.

C2-77. Stochastic diffusion: Source 1's deterministic decay under a stochastic name is Classic-derived.

C2-78. Recent values: Source 1 loses/comment-outs recent min/max infrastructure that Classic keeps live. This is a real port regression relevant to paper-equation comparisons.

C2-79. Runner: Source 1 and Classic have different default loops. Classic starts only diffusion and rent; Source 1 appears to start Hebbian creation/update and excludes Forgetting. Reconciliation should avoid saying Source 1 is a one-to-one runtime port at the runner level.

## Bottom Line

Classic `attention/` is a real ECAN C++ implementation with substantial agent and AttentionBank machinery. It enforces practical STI/LTI accounting, supports AF/WA rent and diffusion, tracks recent STI min/max, and includes tests for the canonical agents. It does not implement the four Source 11 paper equations literally, and it does not close the PLN inference-control integration gap. `metta-attention/` is best read as a partial MeTTa port of this pragmatic Classic ECAN variant, with a few additional regressions and runner differences, rather than a fresh implementation of the 2009 equations.
