Q1. DIRECTOR PHASE MODEL + EVENT HOOKS.
--------------------------------------------------------------------------------

[VERIFIED] Decko Director defines 9 canonical stages across 3 phases (stages.rb:5-7, phases.rb:1-3):
  - **Validation Phase**: :initialize, :prepare_to_validate, :validate. (before_validation hook)
  - **Storage Phase**: :prepare_to_store, :store, :finalize. (around_save hook)
  - **Integration Phase**: :integrate, :after_integrate, :integrate_with_delay. (after_commit hook)

[VERIFIED] Integration events use `rescuing_if_integration` (event.rb:147), ensuring that a failure in one event (e.g., mirror sync) does not roll back the DB transaction or break other events.

VERDICT: [INTEGRATE-STAGE-IS-CANONICAL-HOOK].

Q2. CARD SCHEMA + AUDIT-TRAIL.
--------------------------------------------------------------------------------

[VERIFIED] Card record (schema.rb:71): {id, name, type_id, current_revision_id, trash}.
[VERIFIED] Audit tables:
  - `card_acts` (schema.rb:28): {id, card_id, actor_id, acted_at}.
  - `card_actions` (schema.rb:19): {id, card_id, card_act_id, action_type, draft}.
  - `card_changes` (schema.rb:36): {id, card_action_id, field, value}.

[VERIFIED] `card_actions.draft` usage: Used for `save_draft` which aborts at the `:store` stage (content.rb:8). This means drafts are recorded in actions/changes but the card itself is NOT updated, and Integration phase events (the mirror) DO NOT fire.

[VERIFIED] `ref_type` allowed values (reference/all.rb, chunks):
  - `I`: Inclusion (Nest/Inclusion chunk).
  - `L`: Link (Link chunk [[...]]).
  - `Q`: Query (QueryReference chunk).
  - `P`: Partial (Automatic for virtual card parts).
  - [VERIFIED] URI links are NOT tracked in `card_references` (uri.rb does not inherit from Reference).
