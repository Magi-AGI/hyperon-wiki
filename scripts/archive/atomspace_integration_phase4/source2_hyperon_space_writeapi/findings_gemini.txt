Q1. SPACE WRITE API (CORE GATING QUESTION).
--------------------------------------------------------------------------------

[VERIFIED] The authoritative write API is defined in the `SpaceMut` trait in the new `hyperon-space` crate.

  - `add(&mut self, atom: Atom)`          - hyperon-space/src/lib.rs:223
  - `remove(&mut self, atom: &Atom) -> bool` - hyperon-space/src/lib.rs:241
  - `replace(&mut self, from: &Atom, to: Atom) -> bool` - hyperon-space/src/lib.rs:259

[VERIFIED] `GroundingSpace` (in `lib/src/space/grounding/mod.rs`) implements `SpaceMut`.

[VERIFIED] `DynSpace` is a type-erased wrapper around `Rc<RefCell<dyn SpaceMut>>` (hyperon-space/src/lib.rs:276).

VERDICT: [WRITE-API-CONFIRMED]. Condition (i) is met. Option D is the canonical path.

Q2. SPACEEVENT + SPACEOBSERVER PATTERN.
--------------------------------------------------------------------------------

[VERIFIED] `SpaceEvent` enum (Add, Remove, Replace) - hyperon-space/src/lib.rs:18
[VERIFIED] `SpaceObserver` trait with `notify(&mut self, event: &SpaceEvent)` - hyperon-space/src/lib.rs:56
[VERIFIED] `SpaceCommon` manages a `Vec<Weak<RefCell<dyn SpaceObserver>>>` - hyperon-space/src/lib.rs:94

  - (a) Explicit: `space.common().register_observer(MyObserver{...})` (hyperon-space/src/lib.rs:104).
  - (b) Per-Space: Observers are held in `SpaceCommon`, which is NOT cloned when the space is cloned (hyperon-space/src/lib.rs:125).
  - (c) Synchronous: `notify_all_observers` iterates and calls `notify` immediately (hyperon-space/src/lib.rs:111). A panic in an observer will crash the writer.
  - (d) Synchronous: Runs in the writer's stack frame.
  - (e) Thread-Safe: NO. Uses `Rc` and `RefCell`. The core space implementation is single-threaded.

VERDICT: [OBSERVER-PATTERN-CONFIRMED] [SINGLE-THREADED-LIMITATION].

Q3. PERSISTENCE / DURABILITY MODEL.
--------------------------------------------------------------------------------

[VERIFIED] `space_new` in the C API (c/src/space.rs:69) allows creating a "custom" space by providing a table of function pointers (`space_api_t`). This is the primary hook for external persistence.

[VERIFIED] `PY_SPACE_NO_SUBST_API` (python/hyperonpy.cpp:387) uses this mechanism to wrap Python-defined spaces.

VERDICT: [EXTENSIBLE-VIA-C-API] [NO-NATIVE-RUST-PERSISTENCE-TRAIT].

--------------------------------------------------------------------------------
Q4. CONCURRENCY MODEL.
--------------------------------------------------------------------------------

[VERIFIED] Core implementation uses `Rc<RefCell<...>>` (hyperon-space/src/lib.rs:276).

[INFERRED] For Phase 4 Option D, multi-process Decko workers cannot share a `GroundingSpace` in memory. A central "Space Service" (Python/Rust) with an IPC boundary (gRPC/HTTP) is required, OR each worker maintains a local mirror synced via a shared event bus (like Redis/RabbitMQ).

VERDICT: [INTER-PROCESS-SYNC-REQUIRED].

--------------------------------------------------------------------------------
Q5. DYNSPACE / SPACEMUT / GROUNDINGSPACE TAXONOMY.
--------------------------------------------------------------------------------

[VERIFIED] `hyperon-space` is the new core crate. `lib/src/space/` contains the MeTTa-specific glue and the legacy `GroundingSpace` implementation.

  - GroundingSpace: Concrete in-memory impl.
  - SpaceMut: Trait for mutability.
  - DynSpace: Shared reference wrapper (`Rc<RefCell<dyn SpaceMut>>`).

VERDICT: [REFACTORED-TO-TOP-LEVEL-CRATE].

Q6. PERFORMANCE / SCALE.

[INFERRED] The `AtomIndex` uses a `Trie` (hyperon-space/src/index/trie.rs) for pattern-matched indexing. Throughput is likely high for in-memory operations, but not yet quantified for million-atom scales in this snapshot.

VERDICT: [TRIE-INDEXED-IN-MEMORY].

Q7. ATOM IDENTITY UNDER MUTATION.
--------------------------------------------------------------------------------

[VERIFIED] `replace(from, to)` (hyperon-space/src/lib.rs:259) is implemented as `remove(from)` then `insert(to)`.

[INFERRED] Atom identity is structural. There is no stable UUID for an atom. `SpaceEvent::Replace` carries both the old and new atoms, allowing observers to track the transition.

VERDICT: [STRUCTURAL-IDENTITY] [REPLACE-IS-REMOVE-THEN-ADD].
