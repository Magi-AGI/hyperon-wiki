Q1. MORK KERNEL WRITE-API AUDIT.
    Audit MORK/kernel/src/space.rs (1807 lines) for the canonical write
    surface. Answer:
      (a) What public methods does `pub struct Space` expose for
          atom-mutation? Cite file:line for each (`fn add`, `fn remove`,
          `fn replace`, `fn insert`, `fn delete`, `fn merge`, etc.).
      (b) Does MORK use a trait abstraction analogous to SpaceMut, or is
          the API concrete-only (impl Space {...})?
      (c) What's the data path: `Space.btm: PathMap<()>` is the trie
          backing index. Are writes synchronous to the trie, or buffered?
          Audit the `pub static mut writes: usize` counter — what
          increments it?
      (d) Phase 4/5 implication: if MORK becomes a Phase 5+ candidate
          substrate, its write API must be wrappable as a Hyperon Space.
          Identify the adaptation cost: trait-object vs concrete-struct
          dispatch, identity model translation, observer-pattern bridging.

Q2. MORK SPACE TYPE vs HYPERON SPACEMUT (taxonomy refinement).
    Source 2 Q5 confirmed Hyperon's taxonomy: Space (trait) / SpaceMut
    (trait) / GroundingSpace (concrete) / DynSpace (wrapper). MORK's
    `Space` struct (kernel/src/space.rs:36 confirmed in pre-population)
    is concrete-only.
      (a) Could a `MorkSpace` impl Hyperon's `SpaceMut` trait? What
          would the Atom type translation look like? (Hyperon `Atom`
          enum vs MORK's expression-tree representation.)
      (b) Does MORK's static-counter machinery (`pub static mut
          transitions: usize` etc.) work with Hyperon's per-instance
          observer pattern? Is there an emit-event-on-write shim
          possible?
      (c) Phase 4 (immediate) implication: MORK is NOT a Phase 4
          substrate. Document why.
      (d) Phase 5+ (future) implication: if Phase 5+ wants MORK-backed
          durability, what's the minimum-viable adapter? (Best-guess:
          a Rust crate that imports both `hyperon-space` and `mork`,
          exposes a `MorkBackedSpace: SpaceMut` impl.)

      (b) Does PathMap support MERKLE-IZED trie state (cluster-pilot

      (c) Memmap support: Cargo.toml feature `arena_compact` + MORK's
