Note on terminology (worth flagging early):
  The paper does NOT use the term "FISHGRAM" anywhere. That term is the
  OpenCog-Python implementation name (file: opencog/python/learning/fishgram/
  fishgram.py, sentence-case "Fishgram", per the Pattern class docstring at
  fishgram.py:67 in opencog-singnet @ b31c7e3). The 2013 paper cites Yun Chi's
  external Frequent Subtree Mining software (Chi et al., IEEE TKDE 2005,
  reference [13]) as the algorithmic source. Whether the OpenCog Python
  Fishgram implementation actually uses Yun Chi's algorithm or a different
  re-implementation is an open question for Sources 1 and 2.
